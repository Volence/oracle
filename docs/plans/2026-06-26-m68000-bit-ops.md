## Goal & scope

Add the four single-bit ops on the proven micro-op framework (`m68000::{microop,ea,decode}`). Each op has a
**dynamic** form (bit number in a `Dn`) and a **static** form (bit number in an immediate ext word); each
operates at **32 bits when the destination is a data register** (`Dn`, bit number mod 32) and at **8 bits for
a memory operand** (bit number mod 8):

- **BTST** тАФ test a bit: `Z = NOT(bit)`, **no write** (read-only). FULL source set (Dn, memory, **PC-relative,
  and `#imm`** тАФ it is a read, like TST but wider).
- **BCHG** тАФ test then **toggle** the bit (`operand ^= 1<<pos`). RMW; data-alterable dest.
- **BCLR** тАФ test then **clear** the bit (`operand &= ~(1<<pos)`). RMW; data-alterable dest.
- **BSET** тАФ test then **set** the bit (`operand |= 1<<pos`). RMW; data-alterable dest.

All four affect **only Z** (`Z = NOT(bit-before-modify)`); **N, V, C, X and the SR system byte are all
PRESERVED**. The memory forms are **byte** ops тЖТ **no odd-EA faults** (byte access never address-errors), so
every memory case (incl the plain `(A7)` mode-2 indirect, clean like CLR/TST) is **in scope тАФ no deferral**.

**KEY DATA FACT тАФ each file is 100% PURE for its op** (8065 cases) and MIXES the dynamic + static forms;
classify by opcode. **No MOVEP contamination** (no dynamic mode-001 cases). `tt` (opcode bits 7-6) = 00 BTST /
01 BCHG / 10 BCLR / 11 BSET.

**Deferred (not this push):** shifts/rotates; MUL/DIV; MOVEM/LEA/PEA/LINK/etc; ABCD/SBCD/NBCD; ADDX/SUBX; the
remaining exceptions; MOVEP (the dynamic mode-001 sibling, absent from these files); the integration pivot.

## Recon findings (exhaustive, 0-mismatch verification)

All formulas reproduce the vendored `final` registers + `final.sr` + the per-cycle transaction stream
**bit-exact**. (`vendor/` is gitignored; reproducible from the sha256 manifest.)

- **Flag/value formula (0-mismatch all 4 ops):** `pos = bitnum mod sizebits`; `bit = (operand >> pos) & 1`;
  **`Z = NOT(bit)`**; **only Z changes** (X/N/V/C + the SR system byte all PRESERVED тАФ `ccr = (sr & 0x1B) | (Z
  if bit==0)`, `0x1B = X|N|V|C`). `sizebits` = **32 when the dest/operand is `Dn`** (mode 0), **8 for a memory
  / byte operand**. `bitnum` = **`D[(opcode>>9)&7]`** (dynamic) / **`prefetch[1]`** (static ext word). The
  written value: **BCHG** `operand ^ (1<<pos)`, **BCLR** `operand & !(1<<pos)`, **BSET** `operand | (1<<pos)`;
  **BTST writes nothing**. (BTST verified 0-mismatch on all 7607 Dn+data-memory cases; the PC-rel/`#imm` source
  modes use the same formula and ride the existing `ea_src` machinery.)
- **EA sets.** **BTST (read-only)** = the FULL source set: `Dn` (0), `(An)`/`(An)+`/`тИТ(An)` (2-4), `d16(An)`
  (5), `d8(An,Xn)` (6), `abs.w`/`abs.l` (7/0,7/1), **`d16(PC)` (7/2), `d8(PC,Xn)` (7/3), `#imm` (7/4 тАФ dynamic
  only)** тАФ all via `ea_src` (byte source); NO write, NO `An`-direct. **BCHG/BCLR/BSET (RMW)** = data-ALTERABLE
  dest only: `Dn` (0) + memory (2-6) + `abs.w`/`abs.l` (7/0,7/1) тАФ `ea_dst` byte RMW. (Bit number is a separate
  field, never an EA.)
- **THE LOAD-BEARING TIMING SUBTLETY тАФ register-dest `+2` when `bitnum mod 32 >= 16`** (clean split, 0
  ambiguity). Because the dynamic bit number is a **live `Dn` value**, this is a **DECODE-TIME** decision
  (`decode(regs)` reads it тАФ exactly like Scc's true/false `n2` and DBcc's counter): the register recipe
  appends an `Internal{2}` when `pos >= 16`. **BTST register is FIXED (no `+2` тАФ read-only).** Register cycle
  counts (dynamic / static = dynamic + 4 for the extra bitnum ext word):

  | op | reg, pos<16 | reg, pos>=16 |
  |---|---|---|
  | BTST | 6 / 10 | 6 / 10 (no variance) |
  | BCHG, BSET | 6 / 10 | 8 / 12 |
  | BCLR | 8 / 12 | 10 / 14 |

  i.e. base idle: BTST `n2` (fixed), BCHG/BSET `n2` + (`n2` if pos>=16), BCLR `n4` + (`n2` if pos>=16).
- **Memory timing is FIXED per mode** (byte, bit mod 8 тАФ NO register-style variance). **BCHG/BCLR/BSET memory
  (dynamic)** = the NEG-family byte RMW: `(An)`/`(An)+` 12, `тИТ(An)` 14, `d16(An)` 16, `d8(An,Xn)` 18, `abs.w`
  16, `abs.l` 20. **BTST memory (dynamic)** = those **тИТ4** (no write-back): `(An)`/`(An)+` 8, `тИТ(An)` 10,
  `d16(An)` 12, `d8(An,Xn)` 14, `abs.w` 12, `abs.l` 16; `d16(PC)` 12, `d8(PC,Xn)` 14, `#imm` 10. **Static = dynamic
  + 4** everywhere (the bitnum ext word). All byte тЖТ no odd-EA faults.
- **Static form = the cmpi-style interleave** тАФ the bitnum ext word (`prefetch[1]`) is captured at decode, the
  EA's own ext words follow (a refill shifts the bitnum out first). Mirror `cmpi_recipe` (immediate captured
  first) composed with the `ea_src`/`ea_dst` byte path.
- **Counts.** All four files are **100% in scope** (byte memory тЖТ no odd-EA faults; `(A7)` m2 clean тАФ covered
  like CLR/TST, NOT deferred): **8065 each** (BTST `(A7)`m2 = 150, BCHG = 169, BCLR = 155, BSET = 191 тАФ all
  covered). Total **+32260 тЖТ threshold 494445 тЖТ 526705.**

## Flag & value formulas (verified, 0 mismatches)

Implement as four `AluOp`s sharing one helper (the exec resolves `a` = the operand, `b` = the bit number,
`size` selects mod 32 vs mod 8):

- `let bits = if size == Size::Long { 32 } else { 8 };` `let pos = (b_resolved % bits) as u32;`
  `let bit = (a >> pos) & 1;` `let z = if bit == 0 { CCR_Z } else { 0 };`
  `let ccr = (regs.sr & 0x1B) | z;` (X|N|V|C preserved, Z set from the test)
- value: `AluOp::Btst` тЖТ no write (`Dest::None`); `AluOp::Bchg` тЖТ `a ^ (1<<pos)`; `AluOp::Bclr` тЖТ
  `a & !(1<<pos)`; `AluOp::Bset` тЖТ `a | (1<<pos)`. Returns `(result, ccr)`; the flag write-back path stores
  `result` to `dst` (DataReg full-32 for `Dn`, Scratch(1) for memory) тАФ except BTST (`Dest::None`).

`b` (the bit number) operand: dynamic = `Operand::DataRegFull((opcode>>9)&7)`; static = `Operand::ImmWord`
(`prefetch[1]`). The modulo is applied in the exec (so the full register / full ext word is fed; only the low
bits matter). `a` (the operand) = `Operand::DataRegFull(reg)` for a `Dn` operand, else the read scratch.

## Opcode layouts (classify by opcode тАФ files pure per op)

- **Dynamic** `0000 ddd 1 tt mmm rrr`, mask `opcode & 0xF100 == 0x0100`; `ddd` (bits 11-9) = the bit-number
  `Dn`; `tt` (bits 7-6) = 00 BTST / 01 BCHG / 10 BCLR / 11 BSET; `mmm rrr` = the operand EA. (Mode 001 = MOVEP,
  absent from these files / not decoded.)
- **Static** `0000 1000 tt mmm rrr`, mask `opcode & 0xFF00 == 0x0800`; `tt` (bits 7-6) selects the op; the bit
  number is `prefetch[1]`. (The `*toCCR`/`*toSR` immediate ops are `0x003C/0x007C/...` тАФ bits 8-7 differ; the
  bit-op static block `0x08xx` is disjoint. ANDI/ORI/EORI/SUBI/ADDI/CMPI live at other high-nibble-0 points
  and are not these masks.)

Decode placement: the high-nibble-0 (`0x0xxx`) space already holds CMPI (`0x0Cxx`) and the `*toSR` points; the
bit-op arms (`0xF100==0x0100` dynamic, `0xFF00==0x0800` static) are disjoint. Guard the dynamic RMW arms with
`is_dst_mem_mode`/mode-0; BTST admits the wider source set (mode != 1).

## Recipe shapes (TDD-pinned to the data)

- **BTST** тАФ read-only, `Dest::None`:
  - `Dn` operand (mode 0): `[Prefetch, Alu{Btst, Long, a: DataRegFull(reg), b: <bitnum>, dst: None},
    Internal{2}]` (fixed 6 cyc; no `pos>=16` variance).
  - memory / PC-rel / `#imm` source: the `ea_src` byte read тЖТ `Alu{Btst, Byte, a: <read>, b: <bitnum>, dst:
    None}` (no write). Reuse the `ea_src`/`ea_tst` source machinery (it already covers d16(PC)/d8(PC,Xn)/#imm).
- **BCHG/BCLR/BSET** тАФ a shared `bit_recipe(opcode, AluOp, form)`:
  - `Dn` dest (mode 0): `[Prefetch, Alu{op, Long, a: DataRegFull(reg), b: <bitnum>, dst: DataReg(reg)},
    Internal{base}, (+ Internal{2} when decode computes pos>=16)]`. `base` = `n2` for BCHG/BSET, `n4` for BCLR.
  - memory dest: `ea_dst(&mut buf, mode, reg, Size::Byte, |operand| Alu{op, Byte, a: operand, b: <bitnum>,
    dst: Scratch(1)})` тАФ the NEG-family byte RMW (read тЖТ modify the bit тЖТ write). No register `+2` (byte/mod-8,
    fixed timing).
- **Static form (both):** prepend the bitnum-ext-word capture and route the EA's own ext words after a refill тАФ
  the `cmpi_recipe` interleave (the bitnum is `prefetch[1]` at decode; `b = Operand::ImmWord`).

The **decode-time `pos>=16`** value: `let bitnum = if dynamic { regs.d[(op>>9)&7] } else { regs.prefetch[1] as
u32 }; let pos = bitnum % 32;` append `Internal{2}` to the register BCHG/BCLR/BSET recipe when `pos >= 16`.
`MAX_OPS`/`SCRATCH_SLOTS` unchanged (heaviest: a static abs.l RMW тЙИ 9 ops тАФ measure; bump only if exceeded).

**Anti-cheating (hard rules тАФ a verifier checks these):**
- **SST is ground truth.** Fix the code to match the suite. NEVER weaken an assertion, broaden `covered()` to
  skip cases that should pass, lower the `ran >=` threshold below the true covered count, `#[ignore]`/`#[cfg]`
  -out a test, or comment out an assert. **There is NO deferral this push** тАФ `covered()` admits the full
  in-scope EA set per op (incl `(A7)` mode-2). All files pure single-op (no contaminant).
- **Classify by OPCODE** (`0xF100==0x0100` dynamic / `0xFF00==0x0800` static; `tt` selects the op).
- **Only Z changes** тАФ X/N/V/C **and** the SR system byte are preserved. Z = NOT(the bit BEFORE modify).
- **The register `+2` (BCHG/BCLR/BSET) is a DECODE-TIME `pos>=16` decision** (the dynamic bit number is a live
  `Dn`). Do NOT compute it from a fixed value; read `regs` at decode. BTST register has NO `+2`.
- **Bit width follows the dest** тАФ `Dn` = mod 32 (Long), memory/byte = mod 8. The written register value is the
  FULL 32 bits with one bit flipped; the written memory value is the byte with one bit flipped.
- Both-drivers-agree and snapshot/restore-at-every-bus-boundary stay intact for every new form; the
  determinism gate stays the most-guarded job. No `Rc`/`RefCell`/`unsafe`/`HashMap`/floats in hashed state;
  `MicroState` stays fixed-size bincode. Never touch `../oracle/`.

## Risks

- **The decode-time `pos>=16` register timing** is the load-bearing subtlety (the analog of NEGX's sticky Z /
  Scc's true-false n2). `decode()` reads the live `Dn` (dynamic) or `prefetch[1]` (static), computes `pos =
  bitnum % 32`, and appends `Internal{2}` to the register BCHG/BCLR/BSET recipe iff `pos >= 16`. BTST register
  is FIXED (no +2). Pin BOTH a `pos<16` and a `pos>=16` register anchor per RMW op.
- **Bit width split** тАФ `Dn` operand is 32-bit (mod 32), a memory/imm/PC-rel operand is 8-bit (mod 8). The
  `size` field of the bit `Alu` carries this (`Long` for Dn, `Byte` otherwise). Easy to get the modulo wrong.
- **BTST is read-only with the FULL source set** (incl `d16(PC)`/`d8(PC,Xn)`/`#imm`) тАФ `Dest::None`, reuse
  `ea_src`; do NOT add a write. The RMW trio reuse `ea_dst` byte (read тЖТ bit-modify тЖТ write); Z is from the bit
  BEFORE the modify (the read value).
- **Static form interleave** тАФ the bitnum ext word (`prefetch[1]`) precedes the EA's ext words; mirror
  `cmpi_recipe`. The static timing is dynamic + 4 (the extra refill).
- **`(A7)` mode-2 is COVERED** (byte read/RMW, clean like CLR/TST) тАФ no `reg != 7` carve-out, no parity filter
  (all byte тЖТ no odd-EA faults).
