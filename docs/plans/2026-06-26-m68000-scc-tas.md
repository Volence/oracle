## Goal & scope

Add the two conditional/computed **byte-write** ops on the proven micro-op framework
(`m68000::{microop,ea,decode,bus68k}`). Both are **byte-only**, **data-alterable destination** (`Dn` or
alterable memory), and introduce **one genuinely-new capability each**:

- **Scc.b** — `0101 cccc 11 mmm rrr` (`0x50C0 | cc<<8 | ea`). Write `0xFF` if condition `cc` is true else
  `0x00`, **NO flags**. The condition is resolved at **DECODE time** (like Bcc/DBcc/TRAPV) via the existing
  `condition_true(cc, sr)`. New capability = a **flag-less constant byte write** to a data-alterable dest.
- **TAS.b** — `0100 1010 11 mmm rrr` (`0x4AC0 | ea`). Read the byte → set `N = bit7 / Z = (byte == 0) / V = 0
  / C = 0`, **X preserved** → write `byte | 0x80`. The famous **indivisible read-modify-write**. New
  capability = an **atomic RMW bus cycle** (the single `'t'` transaction the SST stream models) + a unary
  `Tas` ALU op for the `Dn` form.

This is mostly reuse, but **TAS memory is NOT the standard `ea_dst` read+write** (see *Recon findings* — recon
overturned the handoff's "TAS = NEG/NOT read-operand path" assumption). The structurally-new pieces:

| New item | Kind | Used by |
|---|---|---|
| `MicroOp::SetByte { value: u8, dst: Dest }` | no-flag byte constant write (`DataRegLow8` / `Scratch`) | Scc (C0) |
| `AluOp::Tas` | unary: flags from the input byte (`move_flags`+X), write `a \| 0x80` | TAS Dn (C1) |
| `TxKind::Tas` + `Bus68k::tas(addr,fc)->u8` | the atomic RMW bus primitive (ONE `'t'` transaction) | TAS mem (C2) |
| `MicroOp::TasRmw { addr: Operand }` | the atomic byte RMW micro-op (10 cyc, sets flags from the read byte) | TAS mem (C2) |
| `ea_tas` builder | mirrors `ea_dst`'s 7 arms but collapses `Read,PF,Alu,Write` → `TasRmw,PF` | TAS mem (C2) |

**KEY DATA FACT — both files are 100% PURE** (one opcode each, 8065 cases each): every Scc case matches
`op & 0xF0C0 == 0x50C0`, every TAS case `op & 0xFFC0 == 0x4AC0`. **Scc has NO mode-001 (DBcc) contamination**
(DBcc is its own vendored file) and includes all 16 conditions. NO `*I`-style intruder this push.

**Deferred (not this push):** bit ops (BTST/BSET/BCLR/BCHG); shifts/rotates; MUL/DIV; MOVEM/LEA/PEA/LINK/etc;
ABCD/SBCD/NBCD; ADDX/SUBX; the remaining exceptions; the integration pivot. **No intra-family deferral** —
unlike NEG/NEGX/NOT, Scc/TAS **cover the plain `(A7)` mode-2 indirect** (see *Recon findings* — it is clean,
exactly like CLR/TST).

## Recon findings (exhaustive, 0-mismatch verification over all 8065 cases per file)

All formulas reproduce the vendored `final` registers + `final.sr` + the per-cycle transaction stream
**bit-exact with 0 mismatches**. (`vendor/` is gitignored; reproducible from the sha256 manifest.)

- **Scc value** = `0xFF if condition_true(cc, sr_initial) else 0x00` (verified 0 mismatches over all 8065). cc
  = opcode bits 11-8; cc 0 = `T` (ST, always `0xFF`), cc 1 = `F` (SF, always `0x00`) — both legal for Scc
  (unlike Bcc where cc 0/1 are BRA/BSR). **NO flags** (`final.sr == initial.sr` — 0 mismatches). `Dn`-direct
  preserves the upper 24 bits (0 mismatches).
- **Scc timing.** `Dn`: **4 cyc when the condition is FALSE, 6 cyc when TRUE** (a trailing `Internal(2)`; 0
  mismatches — 639 false@4 / 641 true@6 in the sampled Dn set). **Memory** (no true/false difference — the byte
  write happens either way): `(An)` 12, `(An)+` 12, `−(An)` 14, `d16(An)` 16, `d8(An,Xn)` 18, `abs.w` 16,
  `abs.l` 20 — **byte-identical to CLR's RMW timings**. The transaction stream is CLR's exact read-then-write
  RMW (read the EA byte → discard → refill → write the constant).
- **TAS value** = `(byte & 0xFF) | 0x80` (the written byte always has bit 7 set; 0 mismatches, `Dn` + memory).
  `Dn`-direct preserves the upper 24 bits.
- **TAS flags** = `move_flags(byte_read, Byte)` with **X preserved**: `N = bit7(byte_read)`, `Z = (byte_read ==
  0)`, `V = 0`, `C = 0`, X re-injected (`ccr_nz | (sr & CCR_X)`). The flags are on the **value READ** (the
  pre-modify byte), NOT on the written `byte|0x80` (0 mismatches, `Dn` + memory). Structurally the `Not`/logic
  flag shape, but the **written value differs from the flag input**.
- **TAS timing & the atomic `'t'` transaction.** `Dn` = **4 cyc** (`[Prefetch, Alu]`, no bus access). **Memory
  is ONE atomic `'t'` transaction** — `['t', 10, fc=5, addr, '.b', written]` where `written = byte|0x80` — plus
  the prefetch refill(s). 10 cyc = read(4) + the indivisible modify(2) + write(4). The SST stream does **NOT**
  split it into separate `'r'`/`'w'`; it is a single locked bus cycle. Memory timings: `(An)` 14, `(An)+` 14,
  `−(An)` 16, `d16(An)` 18, `d8(An,Xn)` 20, `abs.w` 18, `abs.l` 22 — **= CLR + 2 cyc everywhere** (the atomic
  RMW is 10 cyc vs CLR's read(4)+write(4) = 8). Per-mode structure (TDD-pinned to the stream):

  | Mode | recipe ops | bus stream | len |
  |---|---|---|---|
  | `(An)` / `(An)+` | `[EaCalc?, AdjustAddr?, TasRmw, Prefetch]` | `t, r` | 14 |
  | `−(An)` | `[AdjustAddr, Internal(2), TasRmw, Prefetch]` | `n, t, r` | 16 |
  | `d16(An)` / `abs.w` | `[EaCalc, Prefetch, TasRmw, Prefetch]` | `r, t, r` | 18 |
  | `d8(An,Xn)` | `[EaCalc, Internal(2), Prefetch, TasRmw, Prefetch]` | `n, r, t, r` | 20 |
  | `abs.l` | `[EaCalc(HI), Prefetch, EaCalc(ADDR), Prefetch, TasRmw, Prefetch]` | `r, r, t, r` | 22 |

  i.e. **`ea_dst`'s exact EA-prep + the prefetch(es) BEFORE the operand access, then `TasRmw` replacing
  `Read,Prefetch,Alu,Write`** (the one prefetch `ea_dst` puts between read and write moves to AFTER `TasRmw`).

- **Modes** (both ops). Destinations are **data-alterable**: `Dn` (0), `(An)` (2), `(An)+` (3), `−(An)` (4),
  `d16(An)` (5), `d8(An,Xn)` (6), `abs.w` (7/0), `abs.l` (7/1). **No `An` (1), no PC-relative (7/2, 7/3), no
  `#imm` (7/4)** (none present — a destination must be alterable). Scc excludes mode 001 = **DBcc** (already
  decoded; the DBcc arm runs first / the data-alterable guard excludes mode 1 anyway — and the Scc file
  contains ZERO mode-001 cases). TAS is the `SS == 3` (`0xC0`) sub-case of the `0x4A00` (TST) space — the
  existing TST arm excludes it via `& 0xC0 != 0xC0`.
- **Both are byte-only → NO odd-EA address-error faults** (byte access never faults). No E3/E4 interaction;
  every memory case is a clean access.
- **Plain `(A7)` mode-2 indirect — IN SCOPE (covered, NOT deferred).** Scc memory is CLR's exact `ea_dst` byte
  RMW; CLR/TST cover `(A7)` m2 and PASS (verified: NEG.b `(A7)` m2 is byte-IDENTICAL in structure to CLR.b
  `(A7)` m2 which passes — NEG's deferral was over-conservative, inherited from the old ADD/SUB convention).
  TAS `(A7)` m2 = `[t@A7, prefetch]` (atomic, no increment) — clean. So **cover all data-alterable modes
  including `(A7)` m2**, matching the closest precedent (CLR/TST), maximizing SST coverage per the ground-truth
  principle. (NEG/NEGX/NOT keep their existing deferral; **do NOT touch their prior commits**.) `(A7)+`/`−(A7)`
  byte steps are ±2 (the existing `step_bytes(size, reg)` already handles A7; reuse it in `ea_tas`).

## Flag & value formulas (verified, 0 mismatches)

- **Scc** (decode-time conditional): `let v = if condition_true(cc, sr) { 0xFF } else { 0x00 };` write `v` to
  the byte dest, **touch no flags**.
- **`AluOp::Tas`** (unary, `b` ignored / `Operand::Zero`): `let (_orig, ccr_nz) = move_flags(a, Size::Byte);`
  (N/Z from the input byte, V/C cleared) `let res = (a & 0xFF) | 0x80;` write `res`; `regs.sr = (sr & 0xFF00)
  | ccr_nz | (sr & CCR_X)` (X preserved). The flag input is `a`, the write value is `a|0x80` — they differ.
- **`MicroOp::TasRmw { addr }`** (memory): `let orig = bus.tas(resolve(addr) & ADDR_MASK, regs.fc(false));`
  (Data FC) — `bus.tas` reads `orig`, writes `orig|0x80`, logs ONE `Tas` transaction (value = `orig|0x80`),
  returns `orig`; then `let (_r, ccr_nz) = move_flags(orig as u32, Size::Byte); regs.sr = (sr & 0xFF00) |
  ccr_nz | (sr & CCR_X);` returns **10** cycles. Byte-only → never faults.

## Opcode layouts (classify by opcode — both files pure)

- **Scc** `0101 cccc 11 mmm rrr`, mask `opcode & 0xF0C0 == 0x50C0` (bits 15-12 = 0101, bits 7-6 = 11 isolates
  it from ADDQ/SUBQ which have SS ∈ {00,01,10}). cc = bits 11-8, mode = bits 5-3, reg = bits 2-0. **Place the
  decode arm AFTER the DBcc arm** (`opcode & 0xF0F8 == 0x50C8`, mode 001) so DBcc is consumed first; also guard
  `mode == 0 || is_dst_mem_mode(mode, reg)` (excludes mode 1 = An / DBcc).
- **TAS** `0100 1010 11 mmm rrr`, mask `opcode & 0xFFC0 == 0x4AC0` (`SS == 3` of the `0x4A00` TST space; the
  TST arm already excludes `& 0xC0 == 0xC0`). Guard `mode == 0 || is_dst_mem_mode(mode, reg)`.

## Recipe shapes (TDD-pinned to the data)

- **Scc `<ea>`** — `scc_recipe(opcode, regs)` (condition resolved at decode):
  - mode 0 (`Dn`): `[Prefetch, SetByte { value: v, dst: dn_byte_dest(reg) }, (+ Internal{2} when v == 0xFF /
    condition true)]` → false = 4 cyc, true = 6 cyc.
  - memory: `ea_dst(&mut buf, mode, reg, Size::Byte, |_discarded| MicroOp::SetByte { value: v, dst:
    Dest::Scratch(1) })` — the trailing `Write` stores `Scratch(1)`'s low byte. Byte-only (no `ea_dst_long`).
    Reads the EA byte first (discarded), exactly like `clr_recipe`'s memory arm, but writes `v` with NO flags.
- **TAS `Dn`** — `[Prefetch, Alu { op: AluOp::Tas, size: Byte, a: DataRegLow8(reg), b: Zero, dst:
  DataRegLow8(reg) }]` → 4 cyc.
- **TAS memory** — `ea_tas(&mut buf, mode, reg)` (a dedicated builder mirroring `ea_dst`'s seven arms — same
  `EaCalc`/`AdjustAddr`/`step_bytes`/`push_abs_l_addr`/prefetch-before-the-access — but emitting **one
  `TasRmw { addr }`** in place of the `Read, Prefetch, make_alu, Write` quad, with the trailing `Prefetch`
  AFTER the `TasRmw`). Per the table in *Recon findings*. `addr` = `Operand::AddrReg(reg)` for `(An)`, else
  `Operand::Scratch(EA_SLOT)`.

`MAX_OPS` (20) / `SCRATCH_SLOTS` (10) **unchanged** (heaviest recipe: TAS `abs.l` = 6 ops, Scc `abs.l` = 8 ops;
TAS uses `EA_SLOT` + the abs.l-HI slot, within 10 — measure, bump only if exceeded).

## Risks

- **TAS memory ≠ `ea_dst`** — it is ONE atomic `'t'` bus cycle, not a read+write pair (recon overturned the
  handoff's "TAS = NEG/NOT read-operand path"). The new `TxKind::Tas` / `Bus68k::tas` / `MicroOp::TasRmw` /
  `ea_tas` / runner `'t'`-mapping must all land together (C2). The bus log must contain exactly ONE `Tas`
  transaction per memory TAS, matching the SST `'t'`. Pin the per-mode bus stream from the table.
- **TAS flags are on the INPUT byte, the write value is `input|0x80`** — distinct (unlike `Not`, whose flags
  are on the result). `move_flags(a, Byte)` over the *read* byte gives N/Z; the write is `a|0x80`. Easy to
  conflate — pin a `bit7-already-set` case (N stays 1) and a `byte==0` case (Z=1, write `0x80`).
- **Scc condition at decode time** — `condition_true(cc, sr)` reads the LIVE CCR (like Bcc/DBcc/TRAPV). cc 0/1
  are `T`/`F` (always `0xFF`/`0x00`), both legal. The Dn `Internal{2}`-when-true (6 vs 4 cyc) is the only
  true/false timing difference — memory is condition-independent. Pin both a true and a false Dn case.
- **Scc no-flag write** — use `SetByte`, NOT `AluOp::Move` (CLR uses Move because CLR DOES set flags; Scc does
  not). Verify `final.sr == initial.sr` on every Scc anchor.
- **`(A7)` mode-2 is COVERED** (not deferred) — Scc = CLR's exact path; TAS `(A7)` m2 = `[t@A7, prefetch]`. Do
  NOT add a `reg != 7` carve-out. (`(A7)+`/`−(A7)` byte steps ±2 via the existing `step_bytes` — reuse it.)
