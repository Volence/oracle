# oracle-next — 68000 flow control (Bcc/BRA, BSR, JMP, JSR, RTS, DBcc, RTR)

> **For agentic workers:** REQUIRED SUB-SKILL: `superpowers:test-driven-development` — each behaviour gets a
> *failing test first*. Commits are sequential and dependent (F0→F6); each leaves the tree fully green
> before the next. **GROUND TRUTH IS THE VENDORED SingleStepTests STREAM.** The recipe shapes, cycle counts
> and orderings below were reconnoitred against the real vendored data (an 8-agent recon over the pinned
> `Bcc`/`BSR`/`DBcc`/`JMP`/`JSR`/`RTS`/`RTR`/`Scc` files), but every exact cycle placement and
> bus-transaction ORDER is **TDD-discovered against the data — the SST stream wins, never weaken the
> assertion.** See *Anti-cheating rules*.

## Goal & scope

Add the **flow-control family** on the proven micro-op framework (`m68000::{microop,ea,decode,bus68k,
registers}`): **Bcc** (incl. **BRA** = condition `T`), **BSR**, **JMP**, **JSR**, **RTS**, **DBcc**, and
**RTR**. This is the next genuinely new structural shape: *condition-code evaluation → variable
taken/not-taken paths*, *writing the PC as a branch destination + reloading the prefetch queue*, *DBcc's
decrement-data-reg-and-branch loop*, and *stack push/pop of the 32-bit return address* (reusing the proven
MOVE.l long-access + `AdjustAddr` machinery).

In scope: the seven mnemonics above, **clean (even-target)** cases. The three structural shapes (condition
eval, PC-write + queue reload, stack push/pop). Deferred (stay xfail'd): **address-error exceptions** — any
*odd target / odd EA* (and BSR's `0x61FF` long-displacement form, which traps on the 68000) raises an
address error that pushes a vector-fetch stack frame; these are the existing deferred odd-address class.
Also deferred: **Scc** (a conditional *byte-set*, not flow control — needs flag-less conditional byte write
machinery; next push), the macro-inlined RTC fast path, RTE/TRAP and the exception machinery proper. Never
modify `../oracle/`.

## What flow control reuses vs. adds

**Reuses unchanged:** `Prefetch` (the queue-reload engine — see shape 2), `Read`/`Write` (incl. the long
`Operand::ScratchHi16`/`Scratch` write halves built for MOVE.l, used for the return-address push),
`Internal` (the `n` idle cycles), `AdjustAddr` (SP pre-decrement / post-increment, A7-aware), `EaCalc`
(masked — for the stack address `SP+2`, a real even bus address), `Combine32` (assemble the popped 32-bit
return address / the `abs.l` target, **unmasked**), `Operand::{AddrReg,DispWord,BriefIndex,BriefDisp8,
PcOfExt,Zero,WordStep}`, both drivers, snapshot/restore, the per-cycle transaction gate.

**Adds (small, all introduced in the commit that first uses them):**

| Item | Signature / semantics | First used |
|---|---|---|
| `condition_true(cc: u8, sr: u16) -> bool` | pure helper, all 16 conditions (`T,F,HI,LS,CC,CS,NE,EQ,VC,VS,PL,MI,GE,LT,GT,LE`) against the CCR low byte. **Not a micro-op** — called by `decode`. | F0 |
| `MicroOp::SetPc { value: Operand }` | `regs.pc = resolve(value).wrapping_sub(4)` (**NO mask** — PC stays full 32-bit). 0 cyc, no bus. Primes the two `Prefetch` ops that follow to reload the queue at `value` and leave `pc == value`. | F0 |
| `MicroOp::TargetCalc { base, index, disp, dst }` | `scratch[dst] = resolve(base) + resolve(index) + resolve(disp)` (**NO 24-bit mask** — the unmasked twin of `EaCalc`, because the stored PC / pushed return address is the full 32-bit value). 0 cyc, no bus. | F0 |
| `Operand::BranchDisp8` | `sign_extend8(prefetch[0] & 0xFF)` — the byte-branch displacement (from the **opcode** word, not `prefetch[1]`). | F0 |
| `Operand::PcPlus(u8)` | `regs.pc.wrapping_add(n)` — the return-address base (`n` = instruction byte length: 2/4/6). | F2 |
| `MicroOp::DecrementDnWord { reg: u8 }` | `d[reg] = (d[reg] & 0xFFFF_0000) \| (d[reg].wrapping_sub(1) & 0xFFFF)` — Dn low-word −1, high word preserved, **NO flags** (DBcc counter). 0 cyc. | F5 |
| `MicroOp::LoadCcr { value: Operand }` | `sr = (sr & 0xFF00) \| (resolve(value) as u16 & MASK)` — RTR's popped CCR (the exact `MASK`, 0x1F vs 0xFF, is **TDD-pinned** against RTR data). 0 cyc. | F6 |

No interpreter control-flow change: `exec_one` gains only flat match arms (`SetPc`/`TargetCalc`/
`DecrementDnWord`/`LoadCcr` are each a single straight-line arm). Recipes stay **linear** — variable control
flow is resolved at decode time (shape 1). `MicroState` stays fixed-size bincode. `MAX_OPS`/`SCRATCH_SLOTS`
stay 20/8 unless the JSR `(d8,An,Xn)` / `abs.l` recipe measures larger — bump in the commit that needs it,
documenting the worst recipe.

## The three structural shapes (how flow control maps onto the framework)

### Shape 1 — condition evaluation is resolved at DECODE time, not in the interpreter

`decode()` already takes `&Registers`. For Bcc/DBcc it evaluates the condition (and, for DBcc, the counter)
up front via `condition_true` and emits the **taken** *or* **not-taken** linear recipe directly. The
interpreter stays a flat linear list, so **both drivers + mid-instruction snapshot/restore keep working
unchanged**; variable cycle counts emerge naturally because decode emits different-length recipes. This is
sound because none of these instructions modify the flags they test before deciding (DBcc decrements its
counter *after* the test; the test result is fixed at instruction start). The chosen `MicroState` is
self-contained — an in-flight branch snapshot carries the already-selected recipe.

### Shape 2 — writing PC + reloading the queue reuses `Prefetch`

Verified universal across Bcc/BSR/JMP/JSR/RTS/RTR clean cases: a taken branch / jump / return ends with
`pc = target`, `prefetch = [word@target, word@target+2]` — the **same** "pc = address of `prefetch[0]`"
invariant the sequential recipes already satisfy. `SetPc{target}` sets `pc = target − 4`; the **two
`Prefetch` ops** that follow read at `target` then `target+2` (FC=6 program) and leave `pc = target`. The
−4 is the exact analog of why `Prefetch` reads `pc+4` (the queue is two words ahead). **The PC and the
target are full 32-bit (unmasked)** — e.g. a backward `BSR.w` lands `pc = 0xFFFF_DB42` while the reload
reads at `0xFF_DB42` (`read16` masks the *bus* address only). Hence `SetPc`/`TargetCalc` must NOT mask;
`EaCalc` (which masks) is used only for real even bus addresses like `SP+2`.

The two `Prefetch` ops need not be adjacent — **JSR splits them around the push** (read target → push →
read target+2), while **BSR pushes first then reloads both**. These per-instruction orderings are
TDD-pinned to the data (the `n` idle cycles are NOT in the asserted transaction stream — only reads/writes
are — so idle placement affects only the total cycle count, not the transaction order; cf. the existing
`-(An)` `Internal(2)` rule).

### Shape 3 — stack push/pop reuses the MOVE.l long machinery

The 32-bit return address (BSR/JSR) or popped PC (RTS/RTR) is a long = **two word accesses**. Push: compute
the return address (`pc + N`) into a scratch slot via `TargetCalc{base: PcPlus(N), index: Zero, disp:
Zero}` (unmasked), `AdjustAddr{SP, −4}`, then `Write{hi @ AddrReg(7)}` (value `ScratchHi16(slot)`) and
`Write{lo @ EaCalc(SP+2)}` (value `Scratch(slot)`) — **hi @ SP−4 first, lo @ SP−2** (pinned to data;
distinct from MOVE.l's predec order). Pop: `Read{hi @ SP}` + `Read{lo @ SP+2}` (FC=5 data) → `Combine32`
(unmasked) → target; `AdjustAddr{SP, +4}` (or +6 for RTR).

## Decode layouts

- **Bcc/BRA** `0110 cccc dddddddd` (`0x6xxx`): `cc = (op>>8)&0xF`. `cc==0` → BRA (always taken); `cc==1` →
  **BSR** (separate arm); `cc 2..=15` → conditional. `disp8 = op & 0xFF`; `disp8 == 0` ⇒ **word** form
  (16-bit displacement in `prefetch[1]`), else **byte** form. (`0x60xx`+`0x62..0x6Fxx` are in `Bcc.json`;
  `0x61xx` is in `BSR.json`.)
- **BSR** `0110 0001 dddddddd` (`0x61xx`): byte/word disp like Bcc; `disp8 == 0xFF` ⇒ long-disp form
  (68020) → **always exception on 68000** (xfail).
- **JMP** `0100 1110 11 mmm rrr` (`0x4EC0 | ea`): control EA modes only — `(An)` 010, `(d16,An)` 101,
  `(d8,An,Xn)` 110, `abs.w` 111/000, `abs.l` 111/001, `(d16,PC)` 111/010, `(d8,PC,Xn)` 111/011.
- **JSR** `0100 1110 10 mmm rrr` (`0x4E80 | ea`): same control EA modes.
- **RTS** `0x4E75`. **RTR** `0x4E77`. **DBcc** `0101 cccc 11001 rrr` (`0x50C8 | cc<<8 | reg`): `cc =
  (op>>8)&0xF`, `reg = op & 7`; the 16-bit signed displacement is the extension word (`prefetch[1]`).

## Recipe shapes (orderings TDD-pinned against the vendored data — these are the anchors)

Cycle counts and the canonical clean transaction streams below are from the recon; **pin them with named
anchor cases, fix the recipe to match the data.**

- **Bcc/BRA not-taken (cond false):** `[Internal, Prefetch×k]` where `k` = word count (1 for byte form, 2
  for word form), and the leading idle makes the total. Byte not-taken = **8 cyc** (`62b6`); word not-taken
  = **12 cyc** (`6400`). `pc += 2k`, normal sequential queue advance — **no `SetPc`**.
- **Bcc/BRA taken (cond true):** `[TargetCalc(PcOfExt, Zero, disp), Internal(2), SetPc(scratch),
  Prefetch, Prefetch]` where `disp` = `BranchDisp8` (byte) or `DispWord` (word). **10 cyc** both forms
  (`636a` byte; `6700` word). `TargetCalc` runs **first** (captures `prefetch[1]`/opcode before any refill;
  uses original `pc`).
- **JMP `<control ea>`:** compute the target (per mode) → `SetPc(target)` → `Prefetch, Prefetch`. `(An)` =
  **8 cyc** (no idle); `(d16,An)`/`abs.w`/`(d16,PC)` = **10**; `abs.l` = **12**; `(d8,An,Xn)`/`(d8,PC,Xn)` =
  **14** (6-cyc index idle). Target via `TargetCalc` (unmasked): `(An)` → `SetPc(AddrReg(n))` directly;
  `(d16,An)` → `TargetCalc(AddrReg, Zero, DispWord)`; `(d8,An,Xn)` → `TargetCalc(AddrReg, BriefIndex,
  BriefDisp8)`; `abs.w` → `TargetCalc(Zero, Zero, DispWord)`; `(d16,PC)`/`(d8,PC,Xn)` → base `PcOfExt`;
  `abs.l` → assemble two ext words (the `abs.l` interleave from `ea.rs`, then `Combine32` **unmasked**).
- **BSR (byte/word, even target):** `[TargetCalc(return: PcPlus(N),Zero,Zero), TargetCalc(target:
  PcOfExt,Zero,disp), Internal(2), AdjustAddr(SP,−4), Write(hi @ SP, ScratchHi16(return)), Write(lo @
  EaCalc(SP+2), Scratch(return)), SetPc(target), Prefetch, Prefetch]`. **18 cyc** (`617c` byte, `N=2`;
  `6100` word, `N=4`). Push **hi@SP−4 then lo@SP−2**.
- **JSR `<control ea>` (even target):** like BSR but the reload **splits around the push** —
  `[…compute return & target…, SetPc(target), Prefetch (reads target → prefetch[0]), AdjustAddr(SP,−4),
  Write(hi), Write(lo), Prefetch (reads target+2 → prefetch[1])]`. `(An)` = **16**; `(d16,An)`/`abs.w`/
  `(d16,PC)` = **18**; `abs.l` = **20**; indexed = **22**. Return address = `pc + N` (N = instruction byte
  length; `(An)` 2, `(d16,An)`/`abs.w`/`(d16,PC)` 4, `abs.l` 6, indexed 4). Anchor `4e90 [JSR (A0)]`.
- **RTS (even popped target):** `[Read(hi @ SP, FC=Data), Read(lo @ EaCalc(SP+2), FC=Data),
  AdjustAddr(SP,+4), Combine32(hi,lo → target), SetPc(target), Prefetch, Prefetch]`. **16 cyc** (`4e75`).
- **DBcc (decode-time resolved):**
  - **cond true** → `[Internal, Prefetch, Prefetch]` (pc += 4, 2-word fall-through, **no decrement**). **12
    cyc** (`5dcd`).
  - **cond false, counter not expired** (`Dn.w != 0`) → `[DecrementDnWord(reg), TargetCalc(PcOfExt,Zero,
    DispWord), Internal(2), SetPc(target), Prefetch, Prefetch]` (decrement Dn.w, branch). **10 cyc**
    (`59c8`). Even target in scope; odd target = exception (52 cyc, xfail).
  - **cond false, counter expired** (`Dn.w == 0` → 0xFFFF) → decrement + 2-word fall-through. **Implemented
    for correctness but NOT gate-exercised** — random SST `Dn.w` is ≈never 0 (DBcc data has only lengths
    10/12/52, no expired bucket). Document this honest caveat in the runner.
- **RTR (even popped target):** `[Read(pc_hi @ SP+2), Read(ccr @ SP), Read(pc_lo @ SP+4)]` (the data's
  reordered read stream — pin exactly), `LoadCcr(ccr)`, `AdjustAddr(SP,+6)`, `Combine32(pc_hi,pc_lo →
  target)`, `SetPc(target)`, `Prefetch, Prefetch`. **20 cyc** (`4e77`). Pin the CCR mask to data.

## Clean vs deferred — the `covered()` / `*_in_scope` discriminator

Clean = the computed **target is even** (and, for BSR, the form is byte/word, i.e. `disp8 != 0xFF`). An odd
target/EA raises an address-error exception (a ~14-byte vector-fetch stack frame + a read of the
address-error vector at `0x0C`/`0x0E`) → **xfail**, exactly the existing deferred odd-address class. The
filter computes each target's parity from `ini` exactly as the recipe does:

- **Bcc/BSR/DBcc:** `target = pc + 2 + sign_extend(disp)`; in scope iff `target & 1 == 0`. (Bcc cond
  false → not-taken, always clean; DBcc cond true → no branch, always clean; the parity test applies only
  when the branch would be taken.) BSR also excludes `disp8 == 0xFF`.
- **JMP/JSR:** reuse the shared **`compute_ea`** for the register-file control modes (`(d16,An)`,
  `(d8,An,Xn)`, `abs.w`, `(d16,PC)`, `(d8,PC,Xn)`) and assemble `abs.l` from the two ext words (as the
  existing `even_abs_l` filter does); `(An)` → `addr_reg(reg) & 1`. In scope iff even.
- **RTS/RTR:** the popped 32-bit target is read from `ini["ram"]` at `SP` (RTS) / `SP+2`,`SP+4` (RTR); in
  scope iff even.

`covered()` may ONLY xfail these documented deferred classes — never silently drop a legal even-target case.

## Verification protocol (every commit) & anti-cheating rules

Not done until, from the repo root: `cargo test -p oracle-core --test determinism_gate --test proptests`
green (the most-guarded job), `cargo fmt --all -- --check` clean, `cargo clippy --all-targets -- -D
warnings` clean, `cargo test --workspace` green; committed with a conventional `feat(m68000):` message
ending in `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`. Read `.github/workflows/ci.yml`; run
what CI runs.

**Anti-cheating (hard rules — a verifier checks these):**
- **SST is ground truth.** Fix the recipe/code to match the suite. NEVER weaken an assertion, broaden
  `covered()` to skip cases that should pass, lower the `ran >=` threshold below the true covered count,
  `#[ignore]`/`#[cfg]`-out a test, or comment out an assert. `covered()` may only xfail the documented
  deferred classes (odd-target/odd-EA address errors, BSR `0x61FF`).
- The **both-drivers-agree** and **snapshot/restore-at-every-bus-boundary** assertions stay intact and
  apply to every new mnemonic/shape (add a snapshot/restore anchor per commit — a taken Bcc, a JSR with a
  push, an RTS pop). The determinism gate stays the most-guarded job.
- No `Rc`/`RefCell`/`unsafe`/`HashMap`/floats in hashed state; `MicroState` stays fixed-size bincode;
  introduce vocabulary only in the commit that uses it. Never touch `../oracle/`.

## Risks

- **PC must stay unmasked (full 32-bit); only the bus address masks.** `SetPc`/`TargetCalc` must NOT mask;
  `EaCalc`/`Combine32` keep their existing (masked / unmasked) semantics. Pin with a backward-branch anchor
  whose `final.pc` has high bits set (e.g. `BSR.w` to `0xFFFF_DB42`).
- **The −4 `SetPc` reload-prime** is load-bearing and relies on `Prefetch` reading `pc+4` and shifting the
  queue. Pin with the two-Prefetch reload reads at `target`,`target+2` on a taken Bcc anchor.
- **BSR-vs-JSR push/reload ordering differs** (BSR: push then reload-both; JSR: reload, push, reload) — TDD
  against `617c`/`4e90`; do not assert from memory.
- **JSR/JMP `abs.l`** is the trickiest (two-ext-word interleave + unmasked target) — reuse the `ea.rs`
  `abs.l` pattern; gate on a named clean anchor.
- **DBcc counter-expired path** is correctness-only (absent from data) — implement to spec, document the
  caveat, do not fabricate a gate case.
- **RTR CCR mask + reordered read stream** — pin both exactly to `RTR` data.
- **`MAX_OPS`/`SCRATCH_SLOTS`** — measure the JSR indexed / `abs.l` recipe; bump (documenting the worst
  recipe) if it exceeds 20/8.
