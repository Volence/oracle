## Goal & scope

Add 68000 **exception processing** on the proven micro-op framework (`m68000::{microop,ea,decode,bus68k,
registers}`): the **supervisor-entry / stack-frame-push / vector-fetch / handler-reload** substructure, and the
instructions that use it. This is the next genuinely new structural shape after flow control. In scope (the
owner-approved E0–E6 slice):

- **E0** the exception substructure proven on **TRAP #n** (the cleanest standard 6-byte frame).
- **E1** **RTE** — the exception return (pop SR + PC, restore mode), the inverse of the push.
- **E2** **TRAPV** — a conditional trap resolved at *decode* time (the V flag), like Bcc.
- **E3** the **execution-time address-error abort** + the **group-0 14-byte frame** (the new mechanism).
- **E4** **FLIP the existing odd-address / odd-target xfails to PASSING** across every prior family — the big
  payoff (~45,000 vendored cases become passes for free).
- **E5** **CHK** — a conditional trap that for memory operands is resolved at *execution* time (it reuses the
  E3 abort mechanism); all source modes, both no-trap and trap paths.
- **E6** the privileged SR ops **ANDItoSR / ORItoSR / EORItoSR** (supervisor path) and **RESET**.

**Deferred (not this slice):** DIVU/DIVS (the MUL/DIV arithmetic + the data-dependent restoring-division
cycle model + divide-by-zero — a real time sink with one vendored div0 sample; a later push). The
async-interrupt autovector frame + IPL/mask machinery (shares this substructure but its end-to-end validation
needs the System + scheduler; a later push). The macro-inlined RTC fast path.

**Honest gate caveats (from the recon — document these in the runner, do not fabricate gate cases):**
- **Every vendored case in every file starts in supervisor mode (S=1, T=0).** Therefore the **user→supervisor
  (usp→ssp) stack switch, the T-clear transformation, and the privilege-violation exception are NOT
  gate-testable** by any vendored file — including the *toSR files, which contain only the *supervisor*
  (legal, SR-modifying) path. Implement supervisor entry correctly (set S, clear T, switch A7 to SSP via the
  existing `addr_reg`/`addr_reg_set` S-bit routing) and implement the privilege-violation entry to spec, but
  treat them as **correctness-only, not gate-exercised** (like the DBcc counter-expired path). The S/T/A7
  logic IS exercised structurally by every frame push (just always from supervisor in the data).

## The four structural shapes (how exceptions map onto the framework)

### Shape A — PLANNED exception entry (decode emits the frame recipe directly)

When the trap-vs-not decision is resolvable from the register file at **decode** time (TRAP is unconditional;
TRAPV reads the V flag; CHK with a `Dn`/`#imm` bound), `decode()` emits the full exception-entry recipe as a
**linear** list — exactly the Bcc decode-time-resolution pattern. The recipe is: *capture SR + enter
supervisor → push the standard 6-byte frame → fetch the vector → reload the queue at the handler.*

### Shape B — EXECUTION-TIME abort (a faulting micro-op rewrites its own `MicroState`)

When the fault is only knowable at **execution** time — an **odd word/long address** (address error), a
**CHK bound read from memory** (the trap condition needs the operand), an **RTS/RTR popped target** (read from
the stack) — the interpreter detects it *at the faulting micro-op* and **rewrites the in-flight `MicroState`
into the exception-entry recipe in place** (`self.ops`/`len`/`step=0`, scratch seeded from live state,
`self.cycles` preserved), then both drivers keep calling `exec_one` over the new recipe. This is
hardware-faithful and snapshot-safe (the rewritten `MicroState` is still fixed-size bincode). **No interpreter
control-flow change beyond the in-place recipe swap** — both drivers still loop over `exec_one`; the swap
happens inside the single shared cook, so they cannot diverge.

Why this (vs decode-time resolution of address errors): `abs.l` and the RTS/RTR popped target need a value
`decode` cannot see (the low ext word lives in memory; the return address lives on the stack), and the real
System integration must never let `decode` read memory. Execution-time abort handles **all** modes uniformly
and is the hardware behaviour. **Three properties fall out for free, all confirmed by the recon:**
- **stacked PC = live `regs.pc` at the faulting micro-op** — no special-casing: a taken-branch program fetch
  has already run `SetPc{target}` so `regs.pc == target − 4`; a data fault has `regs.pc == instruction pc +
  2×(prefetches done)`. Both are just `regs.pc`.
- **The BSR/JSR push asymmetry is automatic** — BSR's recipe pushes the return address *before* its faulting
  reload `Prefetch` (SSP ends −18); JSR's faults on the target `Prefetch` *before* its push (SSP ends −14, no
  return push). The existing flow-control recipe ordering produces this with **zero recipe edits**.
- **The live SR is captured at the fault** — so MOVE's data-write fault stacks the SR *with the MOVE's CCR
  already updated*, RTR's odd-pop stacks the *restored* CCR, ADD/SUB RMW stacks the original (the fault is the
  read, before the ALU). All confirmed (0/44965 deviations).

### Shape C — the exception RETURN (RTE), inverse of the push

`RTE` pops the 6-byte frame (SR + 32-bit PC) off SSP, restores the **full SR** (masked to the implemented bits
`0xA71F` — T/S/I/CCR), increments SSP by 6 **while still supervisor**, then reloads the queue at the popped PC
(the reload FC follows the *restored* mode). The companion of RTS, exactly as RTR was. The odd-popped-PC case
is an address error (Shape B), which flips in at E4.

### Shape D — privileged SR ops (decode-time)

`ANDItoSR`/`ORItoSR`/`EORItoSR`/`RESET` are privileged. In supervisor mode (the only vendored path) they run a
small fixed recipe; the privilege check is the S bit at decode time (Shape A would emit the
privilege-violation frame in user mode — correctness-only, not in the data). The `*toSR` ops have a load-bearing
quirk: a mid-instruction SR change that clears S makes the instruction's own re-prefetch run under the **new**
(user) function code — pinned to data via the FC sequence.

## Frame formats (pinned to the vendored data — these are the anchors)

### Standard 6-byte frame (TRAP / TRAPV-trap / CHK-trap / divide-by-zero / privilege-violation)

Pushed to the supervisor stack; `SSP -= 6`. Layout at the post-push SP (`B = SSP_after`):
`SR @ B+0`, `PC-high @ B+2`, `PC-low @ B+4` (the saved return PC, big-endian). **On-bus write order:
`PCL @ B+4` first, `SR @ B+0` second, `PCH @ B+2` third** (the real 68000 microcode order). Then the **vector
fetch** — `read handler-hi @ vector*4`, `read handler-lo @ vector*4 + 2`, **both FC=5 (supervisor-DATA)** — and
the **handler reload** — `read @ handler` (FC=6 supervisor-program), idle `n2`, `read @ handler+2` (FC=6). The
recipe's leading idle (`n4`) and the saved-PC formula are per-exception (pinned below).

- **Pushed SR** = the SR **live at entry**: the *original* SR for TRAP/TRAPV (they touch no CCR); for CHK it
  carries CHK's CCR side-effects (see E5); the running SR after entry is `(live_SR | S) & ~T`.
- **Vectors:** TRAP #n = `32 + (opcode & 0xF)` (addr `(32+n)*4`); TRAPV = 7 (`0x1C`); CHK = 6 (`0x18`);
  divide-by-zero = 5 (`0x14`, deferred); privilege-violation = 8 (`0x20`, correctness-only).
- **Saved PC:** TRAP = `pc + 2`, *no* leading prefetch (the queue is NOT refilled before the push — TRAP's
  first bus event is the `PCL` write). TRAPV-trap = `pc + 2`, *with* a leading prefetch (its first bus event is
  an `FC6 @ pc+4` queue refill). CHK-trap = `pc + instruction_length` (= `regs.pc` after CHK's leading
  operand-read + prefetch). All TDD-pinned.

Anchors: TRAP `4e40` (n=0, vector 0x80, len 34); TRAPV-trap `4e76` V=1 (len 34); TRAPV-no-trap `4e76` V=0
(len 4 — just `r FC6 @ pc+4`).

### Group-0 14-byte address-error frame (vector 3, `0x0C`)

Pushed to SSP; `SSP -= 14` (relative to any instruction-specific stack op already done — see the BSR/JSR/RTS
notes). Layout at `B = SSP_after`: `SSW @ B+0`, `access-addr-hi @ B+2`, `access-addr-lo @ B+4`, `IR @ B+6`,
`SR @ B+8`, `PC-hi @ B+10`, `PC-lo @ B+12` (access-addr and PC are full 32-bit longs). **On-bus write order:
`PCL @ B+12`, `SR @ B+8`, `PCH @ B+10`, `IR @ B+6`, `aLo @ B+4`, `SSW @ B+0`, `aHi @ B+2`** (then the vector-3
fetch @ `0x0C`/`0x0E` FC=5, then the FC=6 handler reload with an `n2` between). The exception **tail** (from
the leading `n4` onward) is byte-for-byte identical across all 44,965 cases and all 13 families; the
per-instruction work is entirely the **prefix** before the `n4`.

Field formulas (0/44965 deviations):
- **SSW** = `(original_opcode & 0xFFE0) | low5`, where `low5 = (read ? 0x10 : 0) | (program ? 0x08 : 0) | fc`,
  `fc` = sv-program 6 / sv-data 5. Observed: `0x1E` program-read (branch/jump/return target fetch), `0x15`
  data-read (incl. the ADD/SUB **RMW read** — RMW always faults on the read, never the write), `0x05`
  data-write (MOVE odd-destination only). The high 11 bits are the **original opcode**, NOT the live/shifted
  `prefetch[0]` (confirmed even for multi-word-EA instructions, e.g. `d06c`: SSW high = `0xD060` = opcode,
  not the shifted ext word).
- **IR @ B+6** = the **original opcode** (latched; survives prefetch shifts).
- **access-addr** = the faulting odd address (full 32-bit): `(An)`→An; `(An)+`→An; `-(An)`→An−size;
  `(d16,An)`→An+sext16; `(d8,An,Xn)`→An+sext8+Xn; `abs.w`→sext16(ext); `abs.l`→(extHi<<16)|extLo (the **odd
  base**; the first long word read faults, acc = base not base+2); program faults → the odd target.
- **stacked PC** = `regs.pc` live at the fault (program fault → target−4; data fault → pc + 2×prefetches).
- **pushed SR** = `regs.sr` live at the fault (original for most; CCR-updated for MOVE write-fault; CCR-restored
  for RTR).

Anchors (fully decoded in the recon, use as TDD pins): `d850` (ADD.w (A0),D4 odd, data-read, len 50);
`d06c` (ADD.w (d16,A4),D0 odd, multi-word-EA, SSW-high test, len 54); `dd56` (ADD.w D6,(A6) odd, RMW read
faults, low5=0x15, len 50); `3c82` (MOVE.w D2,(A6) odd dst, data-WRITE low5=0x05, SR pre-updated, len 50);
`6d25` (Bcc taken odd target, program-read low5=0x1E, stPC=target−4, len 52); `d8b9` (ADD.l (abs.l),D4 odd
base, len 58); `4e90` (JSR (A0) odd, SSP−14 no push, len 50); `6183` (BSR odd, SSP−18 push-then-fault, len
60); `4e75` (RTS odd pop, execution-time, SSP−10, len 58); `4e77` (RTR odd pop, SR=restored CCR, SSP−8, len
62); RTE odd pop `4e73 [RTE] 2` (SSP−8, returning-to-user SSW low5=0x1A, len 62).

## Clean vs deferred — the `covered()` / `*_in_scope` strategy

Every prior family's `covered()` currently **xfails the odd-word/odd-target cases** as the deferred
address-error class. **E4 flips them in:** once E3's abort reproduces the 14-byte frame, the odd cases PASS
unchanged (decode emits the normal recipe; the abort handles the fault), so `covered()` simply **stops
excluding them** — remove the `even()/even_predec()/even_computed()/even_abs_l()` parity filters and the
odd-target filters across ADD/SUB/MOVE/MOVEA/Bcc/BSR/JMP/JSR/RTS/RTR/DBcc (and the RTE/CHK odd cases in their
own files). The recon confirmed **every** odd word/long case in all 13 families produces exactly the 14-byte
vector-3 frame (0 bus-error/vector-2, 0 alternate frames), so there is **no remaining deferred sub-class** to
keep. The only filters that stay: byte accesses never fault (already unfiltered); and any residual non-address
A7 exception cases that were already deferred for other reasons (re-confirm none remain).

**Threshold:** raise the runner `ran >=` to the new measured true count (raised, never lowered). Estimated
~**188,000** (current 78,786 + the 8 new files ≈ 64,520 + the ~44,965 flipped odd cases in existing files);
**measure the exact count at E6 and pin it.**

## Verification protocol (every commit) & anti-cheating rules

Not done until, from the repo root: `cargo test -p oracle-core --test determinism_gate --test proptests`
green (the most-guarded job), `cargo fmt --all -- --check` clean, `cargo clippy --all-targets -- -D warnings`
clean, `cargo test --workspace` green; committed with a conventional `feat(m68000):` message ending in
`Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`. Read `.github/workflows/ci.yml`; run what CI runs.

**Anti-cheating (hard rules — a verifier checks these):**
- **SST is ground truth.** Fix the recipe/code to match the suite. NEVER weaken an assertion, broaden
  `covered()` to skip cases that should pass, lower the `ran >=` threshold below the true covered count,
  `#[ignore]`/`#[cfg]`-out a test, or comment out an assert. After E4, `covered()` may **only** xfail genuinely
  unimplemented classes — it may NOT re-exclude any odd-address case that E3 made coverable.
- The **both-drivers-agree** and **snapshot/restore-at-every-bus-boundary** assertions stay intact and apply to
  every new shape — including the in-place abort (quiesce at the faulting micro-op, snapshot, restore, resume
  into the frame, and get an identical result). The determinism gate stays the most-guarded job.
- No `Rc`/`RefCell`/`unsafe`/`HashMap`/floats in hashed state; `MicroState` stays fixed-size bincode; introduce
  vocabulary only in the commit that uses it. Never touch `../oracle/`.
- The supervisor-entry S/T/A7 transform and the privilege-violation path are implemented to spec but are
  **correctness-only** (no vendored user-mode case) — document this in the runner; do NOT fabricate a gate

- **The in-place `MicroState` rewrite** is the one real interpreter change. Mitigation: it lives inside the
  shared `exec_one` (both drivers get it identically); the rewritten state is ordinary fixed-size bincode; the
  snapshot/restore-across-abort anchor proves serializability. Keep the swap a pure data operation (reassign
  `ops`/`len`/`step`/`scratch`, preserve `cycles`/`opcode`), no early `step += 1`.
- **`MAX_OPS` growth** — the 14-byte frame is the new longest recipe. Prefer `Operand::SpPlus` to keep it
  compact; bump `MAX_OPS` only if the measured op count requires it (document the worst recipe).
- **CHK's two-predicate cycle/flag split** (idle = `Dn>bound?4:6`; N = `Dn<0`) — different predicates; pin both
  to the `4d91`/`4396`/(`Dn<0 && Dn>bound`) anchors, never assume they coincide.
- **`*toSR` mid-instruction FC switch** — the re-prefetch FC depends on the new S bit; pin the `(6,2,2)` vs
  `(6,6,6)` FC sequence to `027c`/`0a7c` switch anchors.
- **PC stays unmasked; only the bus address masks** — `SetPc`/`TargetCalc`/`Combine32` unmasked; the access /
  vector / frame addresses mask via the bus. The address-error `access-addr` field stacks the **full 32-bit**
  faulting address (not 24-bit-masked) — confirmed by the recon (`0x115CED7F`).
- **All data is supervisor-mode** — the usp→ssp switch / T-clear / privilege-violation are not gate-validated;
