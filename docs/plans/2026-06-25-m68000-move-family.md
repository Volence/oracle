## Goal & scope

Add the **MOVE family** ‚Äî `MOVE` (sets flags) and `MOVEA` (to An, no flags) ‚Äî across **byte / word /
long** sizes and **all 12 source EA modes + all alterable-memory destination EA modes**. MOVE is the first
instruction to use a source EA **and** a destination EA in one recipe (EA‚ÜíEA), so it is the structural
proof that `ea_src` and `ea_dst` compose. It also introduces **long-size operands** (`.l` = two word bus
accesses), the deferred piece from the EA push. Builds entirely on the proven micro-op framework
(`m68000::{microop,ea,decode,bus68k,registers}`); reuses `ea_src`/`ea_dst` (already `Size`-aware from C7).

In scope: MOVE.b/.w/.l + MOVEA.w/.l; long operands (two-word accesses, `#imm.l`, full-32 `Dn`); the EA‚ÜíEA
composition; write-only destination (MOVE does NOT read its destination ‚Äî no RMW). Deferred (stay xfail'd):
exceptions / address-errors (odd-word EA), A7 exception cases (the in-scope A7 byte +2 rule already
landed), the macro-inlined RTC fast path. Never modify `../oracle/`.

## What MOVE reuses vs. adds

**Reuses unchanged:** `ea_src` (source operand fetch, all 12 modes), `ea_dst` (destination address
compute ‚Äî already separates address-calc from the access, so MOVE uses the address path and omits the
read), `EaCalc`/`AdjustAddr`, the prefetch-count invariant, the per-cycle transaction gate, both drivers,
snapshot/restore.

**Adds:**
1. **Long-size operands** ‚Äî `Size::Long`. A `.l` operand is **two single-word bus micro-ops** (hi at
   `addr`, lo at `addr+2`), NOT one hidden double access ‚Äî so each word access stays its own transaction
   and its own bus-access quiesce boundary (the ratified granularity). Assemble the two read halves into
   one 32-bit scratch via a `Combine32` op; decompose for writes via `Operand::ScratchHi16` (hi word) +
   `Operand::Scratch` (lo word, Write truncates to the low 16). Brings in `#imm.l` (two extension words,
   assembled like `abs.l` but unmasked) and full-32 `Dn` source/dest.
2. **The MOVE flag op** ‚Äî `AluOp::Move` (size-aware): copies the value and sets **N** (msb), **Z**
   (value==0 at size), **V=0, C=0, X unaffected**. Not arithmetic.
3. **MOVEA** ‚Äî destination mode 1 (An). **No flags.** `.w` **sign-extends** the source word to 32 bits;
   `.l` writes full 32. Byte MOVEA is illegal. Its own decode arm; `AluOp::MoveA` (no flags, sign-extend
   word‚Üílong) writing `Dest::AddrReg`.
4. **Write-only destination path** ‚Äî MOVE composes `ea_src` (read) ‚Üí flag op ‚Üí `ea_dst` address ‚Üí Write
   (no destination read).

## Decode (the MOVE quirks ‚Äî pin with unit tests)

MOVE/MOVEA opcode layout: `00 SS RRR MMM mmm rrr`
- bits 15‚Äì14 = `00`; bits 13‚Äì12 = **size** `SS` (**01=byte, 11=word, 10=long**, 00 = not MOVE);
- bits 11‚Äì9 = **destination register** `RRR`; bits 8‚Äì6 = **destination mode** `MMM` (**swapped** order vs.
  the usual mode/reg);
- bits 5‚Äì3 = source mode `mmm`; bits 2‚Äì0 = source reg `rrr`.

So: `size=(op>>12)&3`, `dst_reg=(op>>9)&7`, `dst_mode=(op>>6)&7`, `src_mode=(op>>3)&7`, `src_reg=op&7`.
`dst_mode==1` ‚áí **MOVEA** (word/long only). Destination modes are alterable: Dn(0) and the alterable-memory
modes (2,3,4,5,6, abs.w, abs.l); **not** PC-relative or `#imm` (not alterable). Source = any of the 12.

## Recipe shapes (orderings TDD-pinned against the vendored MOVE data)

- **MOVE ‚Üí Dn (reg dest):** `ea_src(src, size)` reads the operand; `Alu{Move, size, a:operand,
  dst:DataReg*(dst_reg)}` sets flags + writes the register (low8/low16/full32 by size). Prefetch per the
  word-count invariant; exact placement from SST.
- **MOVE ‚Üí memory:** `ea_src` reads the operand; `Alu{Move, size, a:operand, dst:Scratch(V)}` sets flags +
  parks the value; `ea_dst(dst, size)` computes the address; `Write`(s) at it (one for byte/word, **two**
  for long: hi then lo, order per SST). No destination read.
- **MOVEA ‚Üí An:** `ea_src` reads the operand; `Alu{MoveA, size, a:operand, dst:AddrReg(dst_reg)}` (no
  flags; `.w` sign-extends). No destination memory access.
- **Long operand mechanics:** source long = two `Read{Word}` (addr, addr+2) ‚Üí `Combine32` ‚Üí 32-bit
  scratch; dest long = two `Write{Word}` (`ScratchHi16` then `Scratch`, order per SST). `#imm.l` = capture
  `prefetch[1]` (hi) ‚Üí Prefetch ‚Üí capture `prefetch[1]` (lo) ‚Üí `Combine32`. The MOVE.l `-(An)` and
  long-predec orderings are notoriously quirky ‚Äî **follow the SST stream exactly.**

## Verification protocol (every commit) & anti-cheating rules

Not done until, from the repo root: `cargo test -p oracle-core --test determinism_gate --test proptests`
green (most-guarded), `cargo fmt --all -- --check` clean, `cargo clippy --all-targets -- -D warnings`
clean, `cargo test --workspace` green; and committed with a conventional `feat(m68000):`/`refactor(m68000):`
message ending in `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`.

**Anti-cheating (hard rules ‚Äî a verifier checks these):**
- **SST is ground truth.** Fix the recipe/code to match the suite. NEVER weaken an assertion, broaden
  `covered()` to skip cases that should pass, lower the `ran >=` threshold below the true covered count,
  `#[ignore]`/`#[cfg]`-out a test, or comment out an assert. `covered()` may only xfail the documented
  deferred classes (odd-word address-error, A7 exception cases, not-yet-implemented modes).
- Both-drivers-agree and snapshot/restore-at-every-bus-boundary assertions stay intact and apply to every
  new size/mode (add a snapshot/restore anchor test for a representative MOVE.l mem‚Üímem case in M3).
- No `Rc`/`RefCell`/`unsafe`/`HashMap`/floats in hashed state; `MicroState` stays fixed-size bincode;
  introduce vocabulary only in the commit that uses it. Never touch `../oracle/`.

## Risks

- **Long operand bus ordering** (hi/lo, and the `-(An)`/predec reversal) ‚Äî TDD against `ADD.l` then
  `MOVE.l`; do not assert from memory. `Combine32` must NOT mask to 24 bits (it is an operand value, not
  an address ‚Äî distinct from `EaCalc`).
- **MOVE dest-encoding swap** (reg=11‚Äì9, mode=8‚Äì6) and the size bits (01/11/10) ‚Äî pin with decode unit
  tests before recipes.
- **MOVE is write-only at the dest** ‚Äî do not emit a destination read (that would be RMW); reuse only
  `ea_dst`'s address path.
- **Worst-case recipe length** (MOVE.l mem‚Üímem) ‚Äî measure and size `MAX_OPS`/`SCRATCH_SLOTS` in M3; keep
  fixed-size bincode.
- **Test runtime** ‚Äî MOVE files are large; the SST suite will grow well past 15.8k cases. Acceptable;
  keep the threshold at the true covered count.
