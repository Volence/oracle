## Goal & scope

Add the two 16×16→32 multiply ops on the proven micro-op framework. Both read a **16-bit source** from the
full data-source EA set (the SAME 11 modes as CHK), multiply it by the low word of `Dn`, and write the **full
32-bit product** back to `Dn`:

- **MULU** (`1100 ddd 011 mmm rrr`, opcode & 0xF1C0 == 0xC0C0, opmode 3): **unsigned** `Dn = (Dn & 0xFFFF) *
  (src & 0xFFFF)`.
- **MULS** (`1100 ddd 111 mmm rrr`, opcode & 0xF1C0 == 0xC1C0, opmode 7): **signed** `Dn = sx16(Dn) *
  sx16(src)` (the 32-bit two's-complement product).

The 0xC space's opmode 3/7 is MUL — **disjoint from AND** (opmode 0/1/2/4/5/6; `and_or_in_scope` already
excludes 3/7). Both files are 100% PURE (one opcode each, 8065 cases).

**Flags (both, identical shape):** **N** = bit31(product), **Z** = (product == 0), **V = 0**, **C = 0**,
**X PRESERVED** (re-inject the live X — `move_flags(product, Long)` then `| (sr & CCR_X)`).

**Source EA set** = all 11 data-source modes (An-direct mode 1 is illegal/absent): `Dn` (0),
`(An)`/`(An)+`/`-(An)`/`d16(An)`/`d8(An,Xn)` (2-6), `abs.w`/`abs.l`/`d16(PC)`/`d8(PC,Xn)`/`#imm` (7/0..7/4).
**Odd word EAs are address errors** the E3/E4 abort covers (≈3385 MULU / 3424 MULS cases) — IN SCOPE, no parity
filter, exactly like CHK.

**Deferred (NOT this push):** DIVU/DIVS (+ div0). Their value/V-flag/div0 are solved (in [[recon-mul-div]]),
but the **overflow N/Z** and the **variable Cwik timing** (DIVU normal = 88 + 2·k where k is a function of the
quotient *bit pattern*, not just popcount; DIVS worse; overflow = a flat 10-cyc early-out) require the full
bit-serial ijor microcode for both ops — a dedicated focused push.

## Recon findings (exhaustive, 0-mismatch verification)

All formulas reproduce the vendored `final.d[dn]` + `final.sr` bit-exact on every non-address-error case (4680
MULU + 4641 MULS; the ≈3385/3424 odd-EA cases are address errors handled structurally by E3/E4).

- **Value.** MULU = `(a16 * b16) & 0xFFFF_FFFF` (unsigned). MULS = `(sx16(a) * sx16(b)) & 0xFFFF_FFFF` (signed,
  two's complement). `a` = `Dn & 0xFFFF` (multiplicand), `b` = the source word (multiplier).
- **Flags.** N = bit31(product), Z = (product == 0), V = 0, C = 0, X preserved. (`move_flags(product, Long) |
  (sr & CCR_X)` — the AND/OR/NOT logic-flag shape, but on the full 32-bit product.)
- **Timing (THE interesting part — data-dependent on the SOURCE):** `length = 38 + 2·count + ea_cost(mode)`,
  **0 mismatches across all 4 EA-present forms**. `ea_cost` = the standard word-source EA time (Dn 0, (An) 4,
  (An)+ 4, -(An) 6, d16(An) 8, d8(An,Xn) 10, abs.w 8, abs.l 12, d16(PC) 8, d8(PC,Xn) 10, #imm 4). **`count`
  depends on the SOURCE (the multiplier `b`):**
  - **MULU**: `count = popcount(b & 0xFFFF)` (the number of 1-bits in the 16-bit source).
  - **MULS**: `count = ` the number of `01`/`10` bit-pair transitions in `(b & 0xFFFF) << 1` (a 17-bit value
    with a 0 appended at the LSB) — i.e. `sum over i in 0..15 of (bit(i) != bit(i+1))` of `(b<<1)`. The Booth
    recoding transition count.
- **Bus stream** (TDD-pin): the count-dependent idle comes AFTER the operand read and the prefetch refill.
  Mode-0 `Dn`: `[Prefetch@4, idle@(34 + 2·count)]` (total `38 + 2·count`). A memory source: `[src-read@4,
  Prefetch@4, idle@(34 + 2·count + 0)]` + the EA's own ext-word/idle costs = `38 + 2·count + ea_cost`.
- **Counts.** MULU 8065, MULS 8065 (both 100% in scope; no corrupt entries, no deferral). **Total +16130 →
  threshold 720263 → 736393.**

## The data-dependent-timing mechanism (the one new idea)

Unlike the shift Dn-count (a register read **at decode**), the MUL `count` depends on the **source operand**,
which for a memory mode is **read mid-instruction** (not knowable at decode). So the count-dependent idle is a
**runtime** decision: **the `Alu{Mulu/Muls}` exec arm RETURNS its cycle count** (`34 + 2·count(b_resolved)`)
instead of the usual `0`. `exec_one` already returns the step's cycle cost; the Mul arm resolves `b` (the
source, already in a register / scratch / the immediate), computes `count`, writes the product + flags, and
returns `34 + 2·count`. This uniformly handles all modes (Dn / #imm / memory) with one path — no new `MicroOp`,
no decode-time count. (A dedicated `MicroOp::MulIdle{src}` resolving the scratch source is an equally-valid
alternative if cleaner; the Alu-returns-cycles form is preferred — minimal vocab.) Deterministic + snapshot-safe
(the source is in regs/scratch, captured by the snapshot; both drivers run the Alu identically).

## Recipe shape (TDD-pinned to the data)

Reuse `ea_src` (Word) VERBATIM — the same machinery CHK uses (`chk_recipe`, decode.rs:2860, `ea_src(buf, mode,
reg, Size::Word, |bound| …)`). It already emits the operand read → prefetch refill → the closure op in the
order the MUL stream needs, and covers all 11 source modes incl PC-relative/#imm + the odd-EA address error:

```
fn mul_recipe(opcode, op: AluOp /* Mulu|Muls */) -> MicroState {
    let dn   = (opcode >> 9) & 7;
    let mode = (opcode >> 3) & 7;
    let reg  =  opcode       & 7;
    let mut buf = RecipeBuf::new();
    ea_src(&mut buf, mode, reg, Size::Word, |src| MicroOp::Alu {
        op, size: Size::Word,
        a: Operand::DataRegLow16(dn),  // the multiplicand (low word of Dn)
        b: src,                        // the multiplier (the source word)
        dst: Dest::DataReg(dn),        // the FULL 32-bit product
    });
    buf.finish()
}
```

The `Alu{Mulu/Muls}` exec arm: compute the product (size Long write), set N/Z/V=0/C=0/X-preserved, and
**return `34 + 2·count`** as the step's cycle cost (`count` per op, from `b & 0xFFFF`). `MAX_OPS`/`SCRATCH_SLOTS`
unchanged (MUL ≈ the CHK recipe size).

## Risks

- **The Alu-returns-cycles mechanism** is the one new idea (the first data-dependent cycle count from a value
  not knowable at decode — the memory-source multiplier). Pin a memory-source anchor where the source's
  popcount/Booth count drives the cycle total, and confirm both drivers + snapshot agree.
- **MULS Booth count** ≠ MULU popcount — it's the transition count of `(src<<1)`. Pin an anchor where they
  differ (e.g. source `0x5555` → popcount 8 but Booth 16; source `0xFFFF` → popcount 16 but Booth 1).
- **Odd-EA address errors** (≈3385/3424 per file) are IN scope (E3/E4 word-read abort, like CHK) — no parity
  filter; reuse `ea_src` which already faults on the odd word read.
- **Signed product (MULS)** — sign-extend BOTH operands to 32 bits before multiplying; the low 32 bits of the
  signed product is the result; N = bit31.
