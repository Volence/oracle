# oracle-next Phase 0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:test-driven-development for every
> production change in this plan. Determinism is a single point of failure — every invariant gets a
> failing test *first*. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up the `oracle-core` architecture skeleton (Scheduler + `System` + `Bus`/`BusEvent`
split-borrow + byte-compatible `state_hash`), prove it deterministic with the ported determinism gate,
bring up an instruction-stepped 68000 vertical slice gated on SingleStepTests, and **settle the one
open decision (cycle granularity) empirically** by prototyping `ADD.W D0,(A0)` both instruction-stepped
and FSM-quiesced — then pause for ratification before full opcode coverage.

**Architecture:** One `System` struct owns all memory + chips + the `Scheduler`; the `Scheduler` owns the
sole master clock (`u64` mclk) and one seeded RNG. Chips are generic over `&mut impl Bus` borrowed per
step via split-borrow (no `Rc`/`RefCell`/`unsafe` on the hot path); the bus emits a typed `BusEvent` and
re-entrant cross-chip writes go through one explicit deferred-write seam. The whole machine is plain owned
data deriving `Clone` + bincode, so snapshot/hash/rewind are O(struct).

**Tech Stack:** Rust 1.96, `oracle-core` crate (no I/O, `Send`). bincode 2.x (`Encode`/`Decode`) for
snapshots; hand-rolled SplitMix64 RNG and hand-rolled FNV-1a (no `rand`, no hash crate — zero
dependency-drift risk to determinism). `proptest` + `serde_json` are dev-dependencies (gate runner /
SingleStepTests). No `HashMap` in hashed/serialized state; no floats in hashed state; zero threads in core.

**Scope of THIS push (approved 2026-06-24):** Tasks A–D below — slice + settle cycle granularity, then
**pause** for ratification. Full 68000 opcode coverage, Z80, VDP, `oracle-bus`, and the differential
harness are deferred to later pushes. SingleStepTests are sourced by a pinned-commit fetch + SHA256
(hand-authored subset fallback if the env has no network).

## Verified reference specs (source-confirmed 2026-06-24 against `../oracle/`; do not re-derive)

### `state_hash` byte layout — MUST be byte-compatible with Oracle's `ControlSocket.cpp` `OpStateHash`

Independently cross-checked by two readers of `linux-port/gui/ControlSocket.cpp` (handler ~lines 2374–2419)
and `Devices/315-5313/IS315_5313.h` (size constants).

- **FNV-1a 64-bit.** basis `0xCBF29CE484222325` (`14695981039346656037`), prime `0x100000001B3`
  (`1099511628211`). Per byte: `h ^= (value & 0xFF); h = h.wrapping_mul(PRIME);` (XOR-then-multiply).
  **Every fed value is masked to its low 8 bits.**
- **Five accumulators**, each seeded at the basis: `hv` (vram), `hc` (cram), `hs` (vsram), `hr` (regs),
  `hall` (combined). Each region byte updates *both* its own accumulator and `hall`, in one pass.
- **Region order + fixed sizes** (constants, NOT runtime buffer sizes):
  VRAM `0x10000` (65536) → CRAM `0x80` (128) → VSRAM `0x50` (80) → REGS `24`.
  All ascending index. VDP registers are 8-bit (low byte only). `combined` = one continuous FNV-1a stream
  over the 65768-byte concatenation `VRAM ‖ CRAM ‖ VSRAM ‖ REGS` (no separators/headers/padding).
  `combined` is **not** a hash-of-hashes.
- **Output:** each value is the string `format!("0x{:016X}", v)` (literal `0x`, **uppercase**, zero-padded
  to 16). Result body has **no `ok` key** (Oracle erases it). JSON keys serialize alphabetically.
- **`framebuffer`** is optional (`includeFramebuffer:true`): its own FNV-1a stream over `frame.rgba`
  bytes; **not** folded into `combined`. (Not needed until a renderer exists.)

Reference Rust:
```rust
const FNV_BASIS: u64 = 0xCBF2_9CE4_8422_2325;
const FNV_PRIME: u64 = 0x0000_0100_0000_01B3;
#[inline]
fn fnv1a(h: u64, byte: u8) -> u64 { (h ^ byte as u64).wrapping_mul(FNV_PRIME) }
```

### Determinism gate (`linux-port/harness/determinism_gate.py`, 59 lines)

- **Two fresh instances, sequential** (never concurrent).
- Per instance: `reset {wait:true, run:false}` once (stopped anchor) → loop **120×** of
  `run_frames(1)` then capture `state_hash.combined` → return the 120-string sequence.
- **Pass iff the two 120-element sequences are fully equal** (element-for-element). Exit 0 pass / 1 fail.
- No seed param; determinism comes from a fixed reset anchor + single-threaded device execution
  (`ORACLE_DETERMINISTIC=1` in the C++ build). **Phase 0 ports the LOGIC to a native Rust test** (no bus
  yet); the over-the-bus version arrives with `oracle-bus`.

### Bus protocol (`oracle-bus`, later phase — recorded now for continuity)

Aether JSON-RPC 2.0 over `AF_UNIX`/`SOCK_STREAM` mode 0600, NDJSON framing; `initialize`/`initialized`
handshake (protocolVersion=1, server `"oracle"` v`"2.1-linux"`); live sorted `methods[]`; 52 ops
`emulator/<op>` snake_case. `reset` honors `run` (default true); `run_frames` → `{frames, frame_token}`,
NTSC per-frame quantum `16706515.0` ns. `screenshot` is the **one deliberate divergence**: change disk
path → inline base64 (`docs/foundations.md` lines 79–80).

### Exodus core bootstrap notes (both MIT, Roger Sanders; study + port, re-architect — do NOT fork jgenesis)

- Decode = flat 65536-entry opcode table built from bit-pattern strings; one class per opcode;
  `ExecuteStep()` runs a whole opcode **atomically** and returns ns (the property we re-architect).
- 68000 state to serialize: `D0–D7`, `A0–A6`, `ssp`, `usp`, `pc`, `sr`, prefetch latch
  (`wordIsPrefetched`/`prefetchedWord`/`prefetchedWordAddress`), processor state
  (Normal/Stopped/Exception/Halted), group-0 (bus/address-error) capture latches.
- Bus access: byte/word = 1 access, **long = 2 word accesses** (`addr`, `addr+2`); FC codes classify the
  access. Cycle counts are static per-instruction tables (`ExecuteTime`).
- **Prototype opcode = `ADD.W D0,(A0)` (`0xD150`)**: golden total **12 cycles**, 2 word reads (fetch +
  operand) + 1 word write. Phases: `FETCH → DECODE → EA_READ → ALU → WRITE [→ PREFETCH]`.

## Risks / dependencies (Phase 0)

- **Determinism is a single point of failure.** Any `HashMap` order, float in hashed state, thread, or
  unseeded byte voids `state_hash`. Enforce from commit one (B3).
- **The split-borrow `SystemBus` + deferred-write seam fix the `System` layout.** Designed up front in A5;
  getting them wrong forces a painful refactor or a slide back to `Rc`/`RefCell` (which also breaks `Send`).
- **`state_hash` byte-compat is a hard requirement and a known footgun.** Locked by the verified spec +
  the all-zero golden (A2) + the live-Oracle differential (A7).
- **Network for SingleStepTests** (C1): pinned fetch + checksum; hand-authored subset fallback.
- **Cargo crate fetch:** if the env is offline, minimize deps (manual serialization instead of bincode,
  loop-based property tests instead of proptest) — decided at scaffold time by probing the registry.
