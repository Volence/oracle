## Goal & scope

Add the two 16-bit-divisor → (16-bit quotient, 16-bit remainder) divide ops on the proven micro-op framework,
plus the divide-by-zero trap. Both read a **16-bit source (the divisor)** from the full data-source EA set (the
SAME 11 modes as CHK/MUL) and divide the **full 32-bit `Dn` (the dividend)** by it, writing **quotient → low 16
of `Dn`, remainder → high 16 of `Dn`**:

- **DIVU** (`1000 ddd 011 mmm rrr`, `opcode & 0xF1C0 == 0x80C0`, opmode 3): **unsigned** `q = Dn / src16`,
  `r = Dn % src16`, `Dn = (r << 16) | q`.
- **DIVS** (`1000 ddd 111 mmm rrr`, `opcode & 0xF1C0 == 0x81C0`, opmode 7): **signed** `q = trunc(sx32(Dn) /
  sx16(src))` (truncate toward zero), `r = sx32(Dn) − q·sx16(src)` (**remainder takes the dividend's sign**),
  `Dn = ((r & 0xFFFF) << 16) | (q & 0xFFFF)`.

The 0x8 space's opmode 3/7 is DIV — **disjoint from OR** (opmode 0/1/2/4/5/6). Both files are 100% PURE (one
opcode each, 8065 cases) and 100% in scope. An-direct (mode 1) is illegal/absent.

## Recon findings (exhaustive, 0-mismatch verification — mode-0)

Faithful port of the 68000 division **microcode** (Motorola patent US4325121, reconstructed by
RichardBrown384/68000-microcode-division) + the **Musashi overflow-flag behavior** (the TomHarte suite's
generator). Validated to **0 mismatches** on every mode-0 case (DIVU 1149, DIVS 1220): value, full CCR, the
overflow predicate vs the V flag, and the exact cycle count. The cross-mode overflow-flag rule was confirmed on
all 7 modes (DIVU 4017/4017, DIVS 5295/5295). Recon scaffolding lives in `/tmp/div_recon/` (the validated
Python port + the clean closed-form cycle loops).

### Value (0-mismatch)
- **DIVU:** `q = dividend / divisor` (unsigned), `r = dividend % divisor`. `Dn = (r << 16) | q` (low 16 each).
- **DIVS:** `q = trunc(sx32(dividend) / sx16(divisor))` (C-style truncation toward zero — `(i64)(i32)dd /
  (i16)ds`), `r = sx32(dividend) − q·sx16(divisor)` (so the remainder has the **dividend's sign**).
  `Dn = ((r & 0xFFFF) << 16) | (q & 0xFFFF)`.
- **On overflow: `Dn` is UNCHANGED.** On div0: `Dn` is UNCHANGED (the trap is taken).

### Overflow detection (V = 1) (0-mismatch vs the V flag, all modes)
- **DIVU:** `quotient > 0xFFFF` (equivalently `(dividend >> 16) >= divisor`).
- **DIVS:** `q` out of the signed-16 range, i.e. `q > 0x7FFF || q < -0x8000` (this includes the
  `0x8000_0000 / -1` case, where `q = +0x8000_0000`). Equivalent early-detect form: `abs(dividend) >=
  abs(divisor) << 15` (0-mismatch vs the V flag and vs the early-out timing). **There are ZERO "late overflow"
  cases** in the data — every DIVS overflow is detected early (the flat 16/18-cycle timing below).

### Flags — THE KEY CRACK
The recon-mul-div memory feared the overflow N/Z were "partial internal state, all four combos, no predictor."
**They are not.** They are simply the **initial N/Z carried through** — classic Musashi behavior (the overflow
path sets only V, clears C, and leaves N/Z/X untouched). This is **0-mismatch on 536/536 DIVU + 887/887 DIVS
overflow cases (mode-0)** and **4017/4017 + 5295/5295 across all modes**:

| Outcome | N | Z | V | C | X |
|---|---|---|---|---|---|
| **Normal** | `bit15(quotient)` | `(quotient & 0xFFFF) == 0` | 0 | 0 | preserved |
| **Overflow** | **preserved** | **preserved** | **1** | 0 | preserved |
| **div0 (pushed frame CCR)** | 0 | 0 | 0 | 0 | preserved |

(`preserved` = unchanged from the pre-instruction CCR. For normal, only N/Z change: `ccr = (sr & (CCR_X)) | N |
Z`. For overflow, only V/C change: `ccr = (sr & 0x1F & ~CCR_C) | CCR_V` keeping N/Z/X.)

### div0 trap (vector 5) — pinned to the 1 vendored sample
There is **exactly 1 div0 sample total** (DIVU, `op=0x80ef`, mode 5 `d16(An)`, `len 46`; DIVS has 0). Behavior:
- Trap to **vector 5** (`0x14 = 5*4`), the standard **6-byte frame** (ssp delta 6), reusing the existing
  exception substructure (`EnterException` + `push_standard_frame` + `vector_fetch_and_reload`, exactly like
  CHK's vector-6 trap and TRAP). `Dn` UNCHANGED.
- **The pushed frame CCR is N=0, Z=0, V=0, C=0, X preserved** (this sample has a negative dividend-high word yet
  pushes N=0, so the Moira "N=sign(dividend>>16)" rule is WRONG for this suite — the rule here is simply *clear
  N/Z/V/C, keep X*). Implement to this; pin the 1 case. (Single-sample caveat documented in the runner, like
  the DBcc-expired path.)
- **Saved PC = the live `regs.pc`** at the trap point (= `0xc00` for the sample = the next-instruction address;
  the Alu runs after `ea_src`'s prefetch(es), like CHK). `len = 46` for this mode-5 case.

### Timing (THE other interesting part — the Cwik variable model; 0-mismatch, mode-0)
The variable timing comes from the **faithful 16-step bit-serial microcode** (each microword = 2 cycles). There
is **no simple quotient-bit-count closed form** (popcount fails; the per-iteration cost depends on the partial
remainder's evolution). But the faithful loop reduces to a **clean closed form** (validated 0-mismatch, mode-0).
For memory modes add the standard word-source `ea_cost` (same as MUL: `Dn 0, (An) 4, (An)+ 4, -(An) 6, d16(An)
8, d8(An,Xn) 10, abs.w 8, abs.l 12, d16(PC) 8, d8(PC,Xn) 10, #imm 4`).

**DIVU cycle** (mode-0; `+ea_cost` for memory):
```
if (dividend >> 16) >= divisor { return 10; }            // overflow: flat 10-cyc early-out
let aligned = divisor << 16; let mut work = dividend;
let (mut n_keep, mut n_restore) = (0, 0);
for i in 0..16 {
    let msb = work >> 31;
    work = (work << 1) & 0xFFFF_FFFF;
    if msb != 0 { work = work.wrapping_sub(aligned); }    // mandatory subtract (no extra cost)
    else if work >= aligned { work = work.wrapping_sub(aligned); if i < 15 { n_keep += 1; } }
    else { if i < 15 { n_restore += 1; } }                // restore (q-bit 0)
}
return 76 + 2*n_keep + 4*n_restore;                        // observed range 88..130
```
The `i < 15` guard is load-bearing: the **16th iteration's path is folded into a fixed finalization cost** (the
microcode's last shift goes to DVUM9/DVUMC, not DVUMB/DVUME — no keep/restore microword). Counting all 16 (the
naive bug) over-counts by 2–4 in ~40% of cases.

**DIVS cycle** (mode-0; `+ea_cost` for memory):
```
let q = trunc_div(sx32(dividend), sx16(divisor));
if q > 0x7FFF || q < -0x8000 { return if (dividend as i32) >= 0 { 16 } else { 18 }; }  // overflow early-out
let (add, az) = (abs(sx32(dividend)) as u32, abs(sx16(divisor)) as u32);
let aligned = az << 16; let mut work = add; let mut n_restore = 0;
for i in 0..16 {
    work = (work << 1) & 0xFFFF_FFFF;
    if work >= aligned { work = work.wrapping_sub(aligned); }
    else { if i < 15 { n_restore += 1; } }                // restore-only loop (no msb-mandatory path)
}
let mut base = 110 + 2*n_restore;
base += if (dividend as i32) < 0 { 2 } else { 0 };        // negate-dividend prologue (+2)
base += if (divisor as i16) < 0 { 12 } else if (dividend as i32) < 0 { 14 } else { 10 };  // sign-correction
return base;                                               // observed range 126..152
```
DIVS overflow is a **flat 16 (dividend ≥ 0) / 18 (dividend < 0)** early-out (the `+2` is the negate-dividend
prologue). The DIVS main loop is the simpler restoring division (no mandatory-subtract shortcut), so only
`n_restore` (the q-bit-0 count over the first 15 iterations) varies, plus the dividend-sign prologue and the
3-way sign-correction term.

## The mechanism (reuse the two proven ideas + one trap install)

1. **Source EA + recipe** — reuse the MUL/CHK machinery VERBATIM. A `div_recipe(opcode, op)` mirroring
   `mul_recipe` (decode.rs): `ea_src(&mut buf, mode, reg, Size::Word, |src| MicroOp::Alu{ op, size:
   Size::Word, a: Operand::DataRegFull(dn) /* the full-32 dividend */, b: src, dst: Dest::DataReg(dn) })`, with
   the **`chk_recipe` `#imm` capture-then-idle ordering** for `(mode,reg)==(7,4)` (`EaCalc{ImmWord}→slot` →
   two `Prefetch` → the `Alu` last — NOT literal `ea_src`, whose `#imm` placement mis-orders the idle). All 11
   source modes incl PC-relative/#imm + the odd-EA word-read address error (E3/E4, IN scope). Note `a` is the
   **full 32-bit `Dn`** (the dividend), unlike MUL's `DataRegLow16`.
2. **Data-dependent timing via Alu-returns-cycles** (the MUL mechanism) — the `AluOp::{Divu,Divs}` exec arm
   computes value+flags and **RETURNS its variable cycle cost** (the clean closed forms above), self-booking
   `self.cycles += cost` (the early-return bypasses `exec_one`'s shared tail, exactly like Mulu/Muls). The
   `divisor`/`dividend` are resolved at exec (the source is in a register/scratch/the immediate). One path for
   all modes.
3. **div0 trap install** (the CHK mechanism) — when `divisor == 0`, the arm installs the vector-5 6-byte frame.
   Set the trap CCR (N=0,Z=0,V=0,C=0, X preserved) **before** the frame's `EnterException` captures the SR,
   then rewrite the in-flight `MicroState` into `[Internal{idle}, EnterException, push_standard_frame,
   vector_fetch_and_reload(5*4)]` (mirror `install_chk_trap`/`build_chk_frame` but vector 5 and the div0 CCR;
   saved PC = live `regs.pc`). The leading idle is TDD-pinned so the sole `op=0x80ef` case = `len 46`.

**Implementation choice (impl agent's call):** the above can be one `AluOp::{Divu,Divs}` arm (recommended, mirrors
MUL) that branches normal/overflow (return-cycles) vs div0 (rewrite-MicroState-and-return), OR a dedicated
`MicroOp::DivOp{ dn, signed, src }`. Pick whichever is cleanest against the existing `Alu`/`ChkTrap` patterns;
the gated impl→verify loop settles the exact shape. `MAX_OPS`/`SCRATCH_SLOTS` should be unchanged (the div0
frame ≈ the CHK frame size; confirm).

## Recipe shape (TDD-pinned)
```
fn div_recipe(opcode: u16, op: AluOp /* Divu | Divs */) -> MicroState {
    let dn = ((opcode >> 9) & 7) as u8;
    let mode = (opcode >> 3) & 7;
    let reg  = (opcode & 7) as u8;
    let mut buf = RecipeBuf::new();
    if (mode, reg) == (7, 4) {            // #imm — capture-then-idle (chk_recipe ordering)
        buf.push(MicroOp::EaCalc { base: Operand::ImmWord, index: Operand::Zero, disp: Operand::Zero, dst: DIV_IMM_SRC_SLOT });
        buf.push(MicroOp::Prefetch);
        buf.push(MicroOp::Prefetch);
        buf.push(MicroOp::Alu { op, size: Size::Word, a: Operand::DataRegFull(dn), b: Operand::Scratch(DIV_IMM_SRC_SLOT), dst: Dest::DataReg(dn) });
    } else {
        ea_src(&mut buf, mode, reg, Size::Word, |src| MicroOp::Alu { op, size: Size::Word, a: Operand::DataRegFull(dn), b: src, dst: Dest::DataReg(dn) });
    }
    buf.finish()
}
```

## Risks
- **div0 install from within the Alu arm** — the one structurally new path (the Alu arm has 3 outcomes:
  normal/overflow return-cycles, div0 rewrite-MicroState). Mirror `install_chk_trap`; set the div0 CCR before
  `EnterException`. TDD-pin the sole `op=0x80ef` case (vector 5, 6-byte frame, CCR `0b10000`, saved PC `0xc00`,
  len 46). DIVS has NO div0 sample — implement for correctness, note the absence.
- **The 16th-iteration cycle subtlety** — count keep/restore over iterations 0..14 only (`i < 15`); the 16th is
  folded into the fixed finalization. Counting all 16 over-counts (validated bug). Pin high-cycle anchors.
- **Overflow flag rule** — N/Z/X *preserved*, not computed. Enter an overflow anchor with initial N=1/Z=1 and
  confirm the final has N=1/Z=1/V=1/C=0. (mode-0: DIVU `len 10`, DIVS `len 16|18`.)
- **DIVS sign handling** — trunc toward zero (C `/`), remainder sign = dividend sign; the abs-value loop drives
  the timing; the overflow set is `q∉[-0x8000,0x7FFF]` incl `0x8000_0000/-1`.
- **Odd-EA address errors** (the odd word-read of the divisor) are IN scope via E3/E4, like MUL/CHK — no parity
  filter; `ea_src` already faults on the odd read.
- **`a = DataRegFull(dn)`** (the full 32-bit dividend), NOT `DataRegLow16` (MUL's multiplicand). The product/Dn
  write is `Dest::DataReg(dn)` (full 32).
