# oracle-next — 68000 effective-address machinery (all 12 EA modes + byte size)

> **For agentic workers:** REQUIRED SUB-SKILL: `superpowers:test-driven-development` for every production
> change — each behaviour gets a *failing test first*. Commits are sequential and dependent (C0→C7); each
> must leave the tree fully green before the next begins.
>
> **GROUND TRUTH IS THE VENDORED SingleStepTests STREAM, NOT THIS DOC.** The per-mode micro-op
> orderings below are panel-verified against real clean SST cases, but if any documented ordering or cycle
> placement disagrees with the actual `vendor/ProcessorTests/68000/v1/*.json` data, **the SST data wins —
> fix the *recipe* to match the suite, NEVER weaken the assertion.** See *Anti-cheating rules*.

## Goal & scope

Build the full effective-address machinery: all **12 EA modes** as shared micro-op sub-sequences —
including extension-word reads + their prefetch-queue interaction, and **byte-size accesses** — so any
opcode recipe can use any mode in source or destination position. Vehicle: the existing `ADD.w`/`SUB.w`
families (source-mode word data is already vendored) plus new `ADD.b`/`SUB.b` (byte). Implements the
ratified single-definition hybrid (`docs/decisions/2026-06-24-cycle-granularity.md`).

In scope: 12 source modes; the 7 alterable-memory **destination** modes; word + byte operand sizes; the
two-extension-word **address** assembly for `abs.l`/`#imm.l` (the address, even in a word op). Deferred
(stay xfail'd): long-size **operands** (.l = two word operand reads); exceptions/traps; address-error
(odd EA for a word/long access); A7-specific exception cases (except the in-scope A7 byte +2 step).
Never modify `../oracle/` (study-only).

## Design decision — where address calculation lives (hybrid b+c)

A decode-time **builder** (`crates/oracle-core/src/m68000/ea.rs`) emits, per mode, a sub-sequence of
small shared micro-ops. A single new **`EaCalc`** micro-op deposits the computed effective address into a
**scratch slot** (a visible, serializable mid-instruction value — the charter's introspection goal); a
plain size-tagged `Read`/`Write` then accesses it. Register side effects (`(An)+`/`-(An)`) are a separate
explicit one-shot **`AdjustAddr`** micro-op (the pure `Operand::resolve(&self, &Registers)` cannot mutate
a register; an explicit step is also snapshot-safe and can straddle the `abs.l` prefetch). The hot
interpreter (`exec_one`) stays flat: `EaCalc` is a *fixed* 3-operand `wrapping_add` with **no per-mode
match inside**; the builder (decode time) picks which operands feed it. The only unavoidable runtime
branch — the d8 brief-extension word (data/addr index, word/long size, sign-extension) — is isolated in
one pure `Operand` resolver, not a mode switch.

Rejected: "smart Read" that resolves the address itself (EA never materialized → not snapshot-visible;
read-only resolver can't mutate An; atomic read can't straddle the `abs.l` prefetch). Per-mode `EaCalc`
variants (explodes the `exec_one` match). Duplicate `ReadByte`/`WriteByte` ops (doubles vocabulary on an
orthogonal axis). Stashing the EA in `Registers` (bloats the serializable register file).

## Invariants the builder enforces

1. **Prefetch count == instruction word count.** A k-word instruction emits exactly k `Prefetch`
   micro-ops (so `pc` + the queue end correct). Proof anchors in the existing recipes: `(An),Dn` = 1
   word = 1 `Prefetch`; `#imm,Dn` = 2 words = 2 `Prefetch`. Add a debug assert.
2. **Extension words are consumed from `prefetch[1]` BEFORE the `Prefetch` that shifts them.** `EaCalc`
   (or the `#imm` `Alu`) reads `prefetch[1]` at its sequence position; emit it first. The `abs.l` low
   word is sourced from `prefetch[1]` *after* the interleaved `Prefetch` — **never** from that prefetch's
   bus-return value (that double-counts the queue).
3. **Memory-operand read placement (VERIFIED against SST):** for an `<ea>,Dn` instruction of k words
   with a memory operand, the bus stream is **(k−1) prefetch refills → operand READ → 1 final prefetch**.
   So `(An)`=`[READ,PF]`, `d16(An)`=`[PF,READ,PF]`, `abs.l`=`[PF,PF,READ,PF]`. The operand read is always
   the second-to-last bus event. (The synthesis got this backwards; this is the corrected, verified rule.)
4. One shared `exec_one`; the two drivers (run-to-completion / step-one-micro-op) remain two loops over
   the same `MicroState` — they cannot diverge.
5. `MicroState` stays small, fixed-size, bincode `Encode`/`Decode`. No `HashMap`/floats/`usize`-in-state.
   `MAX_OPS` and `SCRATCH_SLOTS` are consts sized to the worst in-scope recipe.

## Source-read mode table (verified orderings)

`EaCalc(base, index, disp)`; `·` = `Operand::Zero`. `alu_placement`: `First` (#imm), `AfterPrefetch`
(Dn/An), `Last` (memory). `step` = word 2 / byte 1 / **A7 byte 2**.

| Mode (mmm/rrr) | Micro-op sequence | Bus | Cyc |
|---|---|---|---|
| `Dn` 000 | `[Prefetch, Alu(b=DataRegLow16/8)]` | `[PF]` | 4 |
| `An` 001 *(word/long only; LEGAL — not ADDA; no byte; not a dest)* | `[Prefetch, Alu(b=AddrRegLow16)]` | `[PF]` | 4 |
| `(An)` 010 | `[Read(AddrReg), Prefetch, Alu(Scratch)]` | `[READ,PF]` | 8 |
| `(An)+` 011 | `[Read(AddrReg), AdjustAddr(+step), Prefetch, Alu]` | `[READ,PF]` | 8 |
| `-(An)` 100 | `[AdjustAddr(−step), Internal(2), Read(AddrReg), Prefetch, Alu]` | `[READ,PF]` | 10 |
| `d16(An)` 101 | `[EaCalc(AddrReg,·,DispWord), Prefetch, Read(Scratch), Prefetch, Alu]` | `[PF,READ,PF]` | 12 |
| `d8(An,Xn)` 110 | `[EaCalc(AddrReg,BriefIndex,BriefDisp8), Internal(2), Prefetch, Read, Prefetch, Alu]` | `[PF,READ,PF]` | 14 |
| `abs.w` 111/000 | `[EaCalc(·,·,DispWord), Prefetch, Read, Prefetch, Alu]` | `[PF,READ,PF]` | 12 |
| `abs.l` 111/001 | `[EaCalc(·,·,ExtWordHi)→HI, Prefetch, EaCalc(Scratch(HI),·,ExtWordRaw)→ADDR, Prefetch, Read, Prefetch, Alu]` | `[PF,PF,READ,PF]` | 16 |
| `d16(PC)` 111/010 | `[EaCalc(PcOfExt,·,DispWord), Prefetch, Read, Prefetch, Alu]` | `[PF,READ,PF]` | 12 |
| `d8(PC,Xn)` 111/011 | `[EaCalc(PcOfExt,BriefIndex,BriefDisp8), Internal(2), Prefetch, Read, Prefetch, Alu]` | `[PF,READ,PF]` | 14 |
| `#imm` 111/100 | `[Alu(b=ImmWord), Prefetch, Prefetch]` (Alu **First**) | `[PF,PF]` | 8 |

`(An)+`/`-(An)` need no `EaCalc` (the address is just `AddrReg`, read directly; the bump is the explicit
`AdjustAddr`). The `Internal(2)` idle for `-(An)`/d8 modes is non-bus (invisible to the transaction
stream); emit the slot and let the SST `length`/`n`-cycles pin its exact size/position — do not pre-commit
the count from memory.

## Destination EA (alterable-memory modes only)

Modes `(An),(An)+,-(An),d16(An),d8(An,Xn),abs.w,abs.l` (NOT Dn-direct, An, PC-relative, or #imm). The
builder computes the dest address (predec `AdjustAddr` + `EaCalc` emitted **before** the read so read and
write hit the same adjusted address; postinc `AdjustAddr` after the write), then `Read`(old) → refill →
`Alu`(memory is the minuend) → `Write`(new). `Dn,(An)` collapses to today's exact recipe
`[Read(AddrReg), Prefetch, Alu, Write(AddrReg)]` (no `EaCalc`) — guarded by a builder-output-vs-current-
literal diff test in C1. Multi-word **destination** prefetch placement is TDD-pinned per mode against the
`Dn,<ea>` SST cases (the framework expresses whatever stream the suite shows).

## Byte size (C7 — a real bus refactor, not a flag)

`Bus68k` gains `read8(addr,fc)->u8` / `write8(addr,fc,u8)`; `FlatBus` implements them and logs a
**byte-granular** `Transaction` (add `size: Size` to `Transaction`; all existing word literals →
`Size::Word`). `exec_one`'s `Read`/`Write` call read8/write8 vs read16/write16 per `Size`; byte read
zero-extends into scratch. UDS/LDS half: even addr → upper byte, odd addr → lower byte; pin the on-bus
value against a real `ADD.b` case (TDD — do not hardcode the half). SST transaction array is
`[kind, cycles, fc, addr, size_token, value]` (index 4 = `.w`/`.b`, index 5 = value); `expected_transactions()`
reads index 4 into `Transaction.size`. Byte ALU uses 0x80/0x100 flag boundaries + `Dest::DataRegLow8`
(preserve upper 24). Byte EAs may be odd (legal — never odd-filtered). `An`-direct is excluded for byte
(`ADD.b An,Dn` is illegal).

## Edge / xfail handling

- **Address-error filter:** a word/long access to an odd computed EA raises an exception (deferred →
  xfail). `covered()` must compute each mode's EA exactly like the decoder and skip `size==Word && EA&1==1`.
  Use ONE shared pure `compute_ea(opcode, regs_or_ini, size) -> u32` (introduced C4) backing BOTH the
  decoder semantics (or a debug-assert that `EaCalc`'s value equals it) AND `covered()` — a per-mode
  agreement unit test is a **hard gate**, not a mitigation. `abs.l` parity needs the 2nd ext word, which
  is NOT in `prefetch[1]`; read it from `ini["ram"]` at `pc+4`.
- **A7 byte +2 (in scope):** byte `(A7)+`/`-(A7)` step by 2 (A7 stays even). `AdjustAddr` delta =
  `(size==Byte && reg==7) ? ±2 : (size==Byte ? ±1 : ±2)`, written through `addr_reg_set` (ssp/usp-aware).
  `covered()` uses the same step. Odd-A7-word and other A7 exception cases stay xfail.
- Keep every not-yet-implemented mode xfail'd in `covered()` until its commit lands.

## Verification protocol (every commit) & anti-cheating rules

Each commit is **not done** until, from the repo root: `cargo test` (workspace — includes
`determinism_gate`, `proptests`, `singlestep_m68000`, `oracle_differential`) is fully green; `cargo clippy
--all-targets --all-features -- -D warnings` is clean; `cargo fmt --all -- --check` is clean; and the work
is committed with a clear `feat(m68000)`/`refactor(m68000)` message ending in the required
`Co-Authored-By` line. Read `.github/workflows/ci.yml` and run what CI runs.

**Anti-cheating (hard rules — a verifier checks these):**
- **SST is ground truth.** When a case fails, fix the *recipe/code* to match the suite. NEVER weaken an
  assertion, broaden `covered()` to skip a case that should pass, lower the `ran >=` threshold below the
  true covered count, `#[ignore]`/`#[cfg(...)]`-out a test, or comment out an assert.
- The **both-drivers agreement** and **snapshot/restore-at-every-bus-boundary** assertions stay intact
  and apply to every new mode (the no-divergence guarantee). The determinism gate stays the most-guarded
  job.
- `covered()` may only xfail the documented deferred classes (odd-word address-error, A7 exception cases,
  not-yet-implemented modes). It may not silently drop legal, in-scope cases.
- No `Rc`/`RefCell`/`unsafe`/`HashMap`/floats in hashed state; `MicroState` stays fixed-size bincode.
- Never modify `../oracle/`.

## Risks

- `AluPlacement` `First` (for `#imm`) is load-bearing — the first `Prefetch` shifts the immediate out of
  `prefetch[1]`. Pinned by C1's builder-vs-literal diff.
- `compute_ea` drift between decoder and `covered()` — mitigated by the shared helper + hard agreement test.
- The brief extension word (d8 modes) is the most error-prone decode — isolate it in one resolver; TDD
  across both bit11 (W/L) and bit15 (D/A).
- A7 is `ssp`/`usp`, not `a[7]` — `AdjustAddr` must write through `addr_reg_set`.
- Byte on-bus value / UDS-LDS half — TDD against a real `ADD.b` case, never hardcoded.
- Multi-word / PC-rel machinery could ship untested if no vendored case hits it — gate each on a named
  SST anchor.
